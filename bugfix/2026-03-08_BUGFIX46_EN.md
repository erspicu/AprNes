# BUGFIX46: AccuracyCoin P14 APU Register Activation

**Date**: 2026-03-08
**Baseline**: 118/136 → **119/136** (+1)
**Fixed Test**: P14 Test 7 (APU Register Activation) err=6 → PASS

## Root Cause

P14 Test 7 verifies correct behavior when OAM DMA reads from page $40 (APU register space).
When the CPU address bus is in $4000-$401F, DMA can read APU internal registers,
and they are mirrored every $20 bytes ($4036 → $4016, etc.).

Two issues:

### 1. IO_read missing $4017 read handling

`IO_read()` had no case for $4017 (controller 2), causing $4017 reads to
return `cpubus` (write-only default) instead of controller 2 data.
When no controller 2 is connected, it should return D0-D4=0, D5-D7=open bus.

### 2. ProcessDmaRead open bus not updated before bus conflict merge

When DMA reads $4036, it maps to internal $4016 (controller 1), then ANDs the result
with the external bus. The external bus ($4020-$5FFF) returns open bus (cpubus).

Problem: cpubus was not updated after the internal read, still holding the previous
value ($04 from $4015). This caused the controller value $01 to be ANDed with $04 = $00
(should be $01).

Mesen2's `Read()` updates open bus every time, so external reads see the updated value.

## Fix

### IO.cs
```csharp
case 0x4017: return (byte)(cpubus & 0xE0); // Controller 2: D0-D4=0, D5-D7=open bus
```

### MEM.cs (ProcessDmaRead)
In the $4015 and $4016/$4017 cases, update cpubus before the external bus read:
```csharp
cpubus = val; // Update open bus before external read
```
