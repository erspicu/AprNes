## Fix 1: OAM DMA APU Register Activation (P14 test 4, partial)

### Problem
Inside the 2A03, APU registers ($4000-$401F) are only accessible when the 6502 address
bus is in the $4000-$401F range. During OAM DMA, the OAM address bus drives the 2A03
bus, not the 6502. If the 6502 bus was NOT in $4000-$401F when the DMA started, reads
from $4000-$401F during DMA should return open bus (cpubus), not actual APU register values.

This matters because reading $4015 has side effects (clears frame interrupt flag).

### Fix
- `PPU.cs`: `oamDmaExecute()` now takes `cpu6502Addr` parameter, sets `oamDmaApuActive`
  flag based on whether the CPU's address bus was in $4000-$401F at DMA start.
- `IO.cs`: Added check at top of `IO_read()` — during OAM DMA with APU inactive,
  all reads >= $4000 return `cpubus` (open bus) without triggering register side effects.
- `MEM.cs`: `Mem_r()` and `ZP_r()` pass the read address to `oamDmaExecute()`.
- `APU.cs`: Added `oamDmaApuActive` field.

### Limitations
Tests 5-7 require executing code from $3FFE (putting 6502 bus in $4000 range) with DMC
DMA intercepting — extremely complex bus coordination beyond current scope.
