# BUGFIX20: PPU $2007 Read Cooldown

**Date**: 2026-02-22
**Baseline**: 171 PASS / 3 FAIL → **172 PASS / 2 FAIL** (+1 PASS)

## Problem

A dummy read to $2007 during a CPU page-crossing immediately updates the PPU buffer and vram_addr. Real NES PPU has a ~6 PPU dot (~2 CPU cycle) cooldown after a $2007 read, during which consecutive $2007 reads return open bus without triggering a buffer swap or vram_addr increment.

**Failing test**:
- `dmc_dma_during_read4/double_2007_read` — CRC D84F6815 (expected 85CFD627/F018C287/440EF923/E52F41A5)

## Root Cause

### 1. CPU 0xBD (LDA abs,X) page crossing

```asm
ldx #$10
lda $20F7,x   ; $F7+$10 wraps → dummy read $2007 + real read $2107
               ; $2107 also maps to PPU $2007 ($2000-$3FFF mirrors every 8 bytes)
```

CPU reads $2007 twice in two consecutive cycles:
- Cycle 3: `Mem_r($2007)` → buffer swap #1, vram_addr++
- Cycle 4: `Mem_r($2107)` → buffer swap #2, vram_addr++ (should be suppressed)

### 2. Real NES behavior

PPU sets a ~6 PPU dot cooldown after a $2007 read. During the cooldown, subsequent $2007 reads return the open bus value and do not trigger any side effects.

### 3. Mesen2 reference (NesPpu.cpp)

- `_ignoreVramRead = 6` set after normal $2007 read (line 396)
- Decremented by 1 each PPU dot (lines 1498-1503)
- When `_ignoreVramRead > 0`, returns `openBusMask = 0xFF` (full openbus, lines 375-378)
- DMA phantom reads also go through the same ReadRam handler, subject to cooldown

## Fixes

### Fix 1: ppu2007ReadCooldown field (PPU.cs)

Added `static int ppu2007ReadCooldown = 0` field.

### Fix 2: ppu_r_2007() cooldown check (PPU.cs)

```csharp
static byte ppu_r_2007()
{
    if (ppu2007ReadCooldown > 0)
        return openbus; // suppress: no buffer update, no vram_addr increment
    byte result = ppu_read_fun[vram_addr](vram_addr);
    ppu2007ReadCooldown = 6; // 6 PPU dots ≈ 2 CPU cycles
    return result;
}
```

### Fix 3: ppu_step_new() decrement (PPU.cs)

Added per-dot decrement at the start of `ppu_step_new()`:
```csharp
if (ppu2007ReadCooldown > 0) ppu2007ReadCooldown--;
```

### Fix 4: DMC phantom reads bypass cooldown (APU.cs)

DMC DMA phantom reads are each independent bus transactions and should have full side effects. Clear cooldown before phantom reads in `dmcfillbuffer()`:
```csharp
ppu2007ReadCooldown = 0; // DMA phantom reads bypass $2007 read cooldown
```
