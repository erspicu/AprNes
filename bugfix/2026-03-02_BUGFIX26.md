# BUGFIX 26 – D-pad 方向鍵設定 Root Cause：軸值不精確

**日期**：2026-03-02  
**版本**：AprNes  
**分類**：Bug Fix  

---

## 問題

D-pad 在 DirectInput D 模式下，方向鍵完全無法在設定介面中被捕捉（即使診斷工具 JoyTest 確認有 `event_type=0 AXIS` 事件產生）。

---

## 診斷過程

建立 `tools/JoyTest.cs` 獨立診斷工具，確認：
- D-pad D 模式確實產生 `event_type=0 AXIS` 事件 ✓
- 但設定介面欄位完全沒反應

追蹤後發現真正的 root cause。

---

## 根本原因

`JoyPadWayName()` 要求 axis value **精確等於 0 或 65535**：

```csharp
private string JoyPadWayName(string xy_name, int value)
{
    if (xy_name == "X")
    {
        if (value == 0) return "LEFT";
        if (value == 65535) return "RIGHT";
    }
    if (xy_name == "Y")
    {
        if (value == 0) return "UP";
        if (value == 65535) return "DOWN";
    }
    return "";  // ← 任何其他值都返回空字串
}
```

原本的 `AXIS_NOISE` 過濾只處理**靠近中心**（±256 of 32767）的雜訊，但 D-pad 極值（最小/最大）的軸值若有偏差（例如 `1`、`100`、`65434` 等），`JoyPadWayName` 返回 `""` 導致：

1. **設定時**：`Setup_JoyPad_define()` 因 `name == ""` 提前返回，欄位無顯示，keymap 未寫入
2. **遊戲時**：gaming lookup key 包含空字串，永遠查不到 keymap → 方向鍵無反應

可能觸發的情境：
- `DIPROP_RANGE` SetProperty 呼叫失敗（裝置不支援，靜默忽略），軸仍使用硬體原始範圍（非 0~65535）
- 部分手把製造商的 D-pad 不回傳精確極值

---

## 修復

在 `joystick.cs` 新增 `SnapAxisDigital()`，將所有軸值 snap 到三個離散值：

```csharp
static int SnapAxisDigital(int v)
{
    if (v < 16384) return 0;       // 低半段 → 負方向
    if (v > 49151) return 65535;   // 高半段 → 正方向
    return 32767;                   // 中間 → 中性
}
```

套用範圍：
- **DirectInput** `lX`/`lY`（在 AXIS_NOISE 中心過濾後再 snap）
- **XInput** analog stick X/Y（在 deadzone 過濾後再 snap）
- **POV hat → XY** 本身已輸出精確值，不需修改

修改前後比較：

| 手把回報值 | 修改前 | 修改後 |
|----------|--------|--------|
| 0 | "UP" ✓ | "UP" ✓ |
| 1 | "" ✗ | "UP" ✓ |
| 32767 | 中性，略過 ✓ | 中性，略過 ✓ |
| 65534 | "" ✗ | "DOWN" ✓ |
| 65535 | "DOWN" ✓ | "DOWN" ✓ |
| -1000（DIPROP_RANGE 失敗） | "" ✗ | "UP" ✓ |
