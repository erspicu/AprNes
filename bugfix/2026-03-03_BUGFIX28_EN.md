## 1. Sync Interop Optimization (FPS 70 â†’ ~100)

### Problem
- JS used `invokeMethodAsync` (returns a Promise), generating microtask scheduling overhead every frame
- `JS.InvokeVoidAsync("drawFrame", rgba)` serialized a 245KB byte[] as Base64 JSON, then passed it to JS for decoding
- `OnFrame()` was `async Task`, incurring async state machine allocation cost every frame

### Fix

**`nesInterop.js`**
- `invokeMethodAsync` â†’ `invokeMethod` (synchronous direct call, no Promise)

**`NesJsInterop.cs`** (new file)
```csharp
[JSImport("globalThis.nesInterop.drawFrameUnmarshalled")]
internal static partial void DrawFrame(byte[] pixels);
```
Uses `[JSImport]` synchronous call to replace `JS.InvokeVoidAsync`

**`NesFrameStep.cs`**
```csharp
static readonly byte[] _rgbaBuffer = new byte[256 * 240 * 4];
```
Reuses a static buffer, eliminating per-frame GC pressure

**`Home.razor`**
- `OnFrame()` changed to `void` (synchronous), eliminating async state machine overhead

### Result
| State | FPS (uncapped mode) |
|-------|---------------------|
| Before fix | ~70 |
| After fix  | ~100 |

## 2. Game Restart Loop Double-Run Problem

### Problem
After stopping a game and loading a new one, the old JS loop was not truly stopped.  
Once `_romLoaded` was set to `true` by the new game, the old loop resumed executing frames, causing:
- Doubled screen FPS (when uncapped)
- Severe audio/video desync

### Root Cause
`startLoop` returned a `{ stop() }` object, but the Blazor side never stored and called `stop()`.

### Fix
Used a **generation counter** in `nesInterop.js` to automatically invalidate old loops:

```javascript
let loopGeneration = 0;

function startLoop(dotNetRef) {
    loopGeneration++;              // invalidate all old loops
    const myGen = loopGeneration;
    nextTime = 0;                  // reset audio timeline
    function loop() {
        if (loopGeneration !== myGen) return; // old loop, stop
        dotNetRef.invokeMethod('OnFrame');
        ...
    }
}

function stopLoop() { loopGeneration++; }
```

`Home.razor` `StopGame()` changed to async, calling `nesInterop.stopLoop()` first:

```csharp
async Task StopGame() {
    NesCore.exit = true;
    await JS.InvokeVoidAsync("nesInterop.stopLoop");
    _romLoaded = false;
    ...
}
```

## 3. HTML5 Gamepad API Support

### Feature
- Supports XInput (Xbox series) and DirectInput gamepads
- Standard gamepad button mapping:

| NES | Gamepad |
|-----|---------|
| A | buttons[0] (A/â—‹) |
| B | buttons[1] or buttons[2] (B/X) |
| Select | buttons[8] |
| Start | buttons[9] |
| D-pad | buttons[12-15] or left stick axes[0/1] |

- Updates every frame in `OnFrame()` using delta detection (only calls Press/UnPress on state change)
- Listens for `gamepadconnected` / `gamepaddisconnected` events to show connection status in real time

### Note
Browser security restriction: after plugging in a gamepad, you must **press any button** first before the `gamepadconnected` event fires.
