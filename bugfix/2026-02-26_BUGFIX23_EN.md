## Issue 1: AprNesAOT Window Blank — No UI Elements

**Symptom**:
Window opens (title bar normal), but window content is completely blank — no menu bar, no hint text, no interaction entry points.

**Root Cause**:
The initial version of `Program.cs` only created an empty window without adding a Win32 menu bar (`CreateMenu`) or hint text (`DrawTextW`).

**Fix**:
- Added a complete menu bar: `File(F)` / `Options(O)` / `Language(L)` / `Help(H)`
- Added `OpenRomDialog()` (using `GetOpenFileNameW`)
- `WM_PAINT` displays hint text when no ROM is loaded

## Issue 2: App Crashes a Few Seconds After Launch (EntryPointNotFoundException)

**Symptom**:
```
Unhandled exception. System.EntryPointNotFoundException:
Unable to find an entry point named 'SetBkMode' in DLL 'user32.dll'.
```
App crashes within seconds of startup (on first `WM_PAINT` with no ROM loaded).

**Root Cause**:
`SetBkMode` and `SetTextColor` are **GDI functions** located in `gdi32.dll`, but the P/Invoke declarations incorrectly specified `user32.dll`:

```csharp
// ❌ Wrong
[DllImport("user32.dll")] static extern nint SetBkMode(nint hdc, int mode);
[DllImport("user32.dll")] static extern uint SetTextColor(nint hdc, uint color);
```

**Fix**:
```csharp
// ✅ Correct
[DllImport("gdi32.dll")] static extern nint SetBkMode(nint hdc, int mode);
[DllImport("gdi32.dll")] static extern uint SetTextColor(nint hdc, uint color);
```

**Affected file**: `AprNesAOT/Program.cs`
**Fix commit**: `2d729a1`

## Feature Addition: AprNesAOT Full UI Port (matching original WinForms functionality)

The original AprNes uses WinForms, which cannot be used with Native AOT. The following features were re-implemented using pure Win32 P/Invoke:

### Menu Structure

```
File(F)
  ├─ Open ROM (O)
  ├─ ─────────────
  ├─ Soft Reset (S)       ← enabled only after ROM loaded
  ├─ Hard Reset (H)       ← enabled only after ROM loaded
  ├─ ─────────────
  └─ Exit (X)

Options(O)
  ├─ Sound (S)            ← checkmark toggle
  ├─ Volume (V) ►
  │    ├─ 10% / 30% / 50% / 70% / 90% / 100%  (Radio)
  ├─ ─────────────
  └─ Limit FPS (F)        ← checkmark toggle

Language(L)
  └─ (dynamically generated from AprNesLang.ini, checkmark on current language)

Help(H)
  ├─ ROM Info (I)         ← enabled only after ROM loaded
  └─ About (A)
```

### Other Features

| Feature | Description |
|---------|-------------|
| FPS title bar display | `AprNes AOT  [rom name]  FPS: 60`, updated by WM_TIMER every second |
| Settings read/write | Reads/writes `AprNes.ini` (keys/sound/volume/FPS limit/language) |
| Key mapping | Loads `_keyMap[8]` from ini, same format as original |
| ROM info dialog | Shows Mapper / PRG / CHR / Mirror / Battery |
| Soft reset | `NesCore._event.Reset()` → `SoftReset()` → `Set()` |
| Hard reset | Stop emulation thread → `NesCore.init()` reinitialize → restart |
| Drag-and-drop ROM loading | `WM_DROPFILES` (preserves original functionality) |
| Language switching | Writes to ini after selection, prompts for restart |

### New NesCore Public Properties (Main.cs)

```csharp
static public int  RomMapper   => mapper;
static public int  RomPrgCount => PRG_ROM_count;
static public int  RomChrCount => CHR_ROM_count;
static public bool RomHorizMirror => (ROM_Control_1 & 1) == 0;
static public bool LimitFPS = false;
```
