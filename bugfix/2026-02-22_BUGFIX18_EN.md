# BUGFIX18: CPU Interrupt Timing — penultimate-cycle IRQ + NMI deferral + DMA alignment

**Date**: 2026-02-22
**Baseline**: 165 PASS / 9 FAIL → **169 PASS / 5 FAIL** (+4 PASS)

## Problems

- `cpu_interrupts_v2/2-nmi_and_brk` — rows 9-10: NMI fires before SEC instead of after
- `cpu_interrupts_v2/4-irq_and_dma` — IRQ after DMA fires at wrong instruction
- `cpu_interrupts_v2/5-branch_delays_irq` — test_jmp baseline fails (3-cycle JMP penultimate wrong)
- `cpu_interrupts_v2/cpu_interrupts` — merged test (due to sub-test 2,4,5 failures)

## Root Causes

### 1. irqLineAtFetch too early for 3+ cycle instructions (tests 4, 5)

`irqLineAtFetch` samples IRQ line state after the opcode fetch tick. This is correct for 2-cycle instructions (tick 1 = penultimate), but is too early for 3-cycle JMP (penultimate = tick 2) and OAM DMA (517 ticks, penultimate = tick 516).

### 2. NMI fires too early after BRK/IRQ/NMI sequence (test 2)

NMI edge occurs during BRK cycles 6-7 → nmi_delay → promote → nmi_pending → run() immediately fires NMI (before SEC in IRQ handler). Real NES: interrupt sequence does not do NMI polling; NMI fires on the first instruction of the handler.

### 3. Branch taken-no-cross IRQ detection error (test 5)

Taken branch with no page cross has 3 ticks, but penultimate should be tick 1 (not tick 2). Tick 3 (extra tick) does not participate in IRQ polling.

### 4. IRQ timing offset after OAM DMA (test 4)

During DMA, tick() continuously updates irqLinePrev/irqLineCurrent, causing irqLinePrev after DMA to reflect the DMA end state rather than the pre-DMA state. Additionally, DMA was missing an alignment cycle (513 → 514 ticks).

## Fixes

### Fix 1: irqLinePrev/irqLineCurrent per-tick tracking (MEM.cs)

Track previous/current IRQ line state at end of tick(). After instruction completes, irqLinePrev naturally holds the IRQ state from the penultimate tick, applicable to all instruction lengths.

```csharp
// End of tick()
irqLinePrev = irqLineCurrent;
irqLineCurrent = statusframeint || statusdmcint || statusmapperint;
```

### Fix 2: NMI deferral after interrupt sequences (Main.cs)

Added `nmi_just_deferred` flag. After a BRK/IRQ/NMI interrupt sequence completes, if nmi_pending is set, defer NMI by 1 instruction before firing.

### Fix 3: Branch irqLinePrev save/restore (CPU.cs)

In the 8 branch instructions, save irqLinePrev on taken branch, and restore it if there is no page cross (extra tick does not affect IRQ polling).

### Fix 4: OAM DMA isolation + alignment (PPU.cs)

- Save irqLinePrev before DMA, restore after
- Added alignment tick (1 halt + 1 align + 256×2 R/W = 514 ticks)

### Fix 5: Cleanup

- Removed `irqLineAtFetch` (CPU.cs)
- Removed `oam_dma_occurred` (PPU.cs + Main.cs)
- Removed `branch_taken_no_cross` (CPU.cs + Main.cs)
