# BUGFIX21: TestRunner --pass-on-stable

**Date**: 2026-02-22
**Baseline**: 172 PASS / 2 FAIL → **174 PASS / 0 FAIL** (+2 PASS) — All tests passing!

## Problem

`read_joy3/count_errors` and `count_errors_fast` exit silently (exit code 0) on success, printing no "Passed" text. TestRunner's `--wait-result` waits for "Passed"/"Failed" text and reports FAIL on timeout when it cannot find either.

**Failing tests**:
- `read_joy3/count_errors` — Timeout (no $6000 protocol, result=0xFF)
- `read_joy3/count_errors_fast` — Same

## Root Cause

### blargg test framework exit mechanism

```
exit code 0 → silent exit (prints nothing)
exit code 1 → prints "Failed"
exit code ≥2 → prints "Error N"
```

There is no "exit code 0 → print Passed" mechanism. `tests_passed` is a separate function that is not called automatically.

### count_errors logic

- Starts DMC playback, calls `read_joy` (4-read voting algorithm) 1000 times
- `read_joy` returns 0 (no key press) = normal → continue
- `read_joy` returns non-zero = compensation failure → `test_failed` → prints "Failed"
- Main loop ends → `rts` → exit code 0 → **silent exit**

### count_errors_fast logic

- Starts DMC playback, calls `read_joy_fast` (single read, no compensation) 1000 times
- Counts errors, prints "Errors: N/1000"
- Never prints "Passed" or "Failed"

## Fixes

### Fix 1: TestRunner --pass-on-stable (TestRunner.cs)

Added `--pass-on-stable` parameter. In screen stability detection:
- Screen stable + has "Failed" → **FAIL**
- Screen stable + no "Failed" → **PASS** (because the test exits silently on success, the screen stops changing)

```csharp
if (!earlyPass && !earlyFail && passOnStable)
{
    if (NametableContains("Failed") || NametableContains("FAILED"))
        earlyFail = true;
    else
        earlyPass = true;
}
```

### Fix 2: Test script updates

`run_tests.sh` and `run_tests_report.sh` add `--pass-on-stable` for count_errors/count_errors_fast.
