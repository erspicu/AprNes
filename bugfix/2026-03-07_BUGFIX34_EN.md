# BUGFIX34: $2007 rendering increment, $2004 read/write during rendering

## Date: 2026-03-07

## Baseline: 174 PASS (legacy tests), 107/136 AccuracyCoin PASS (after BUGFIX33)

## Fix Details

### 1. $2007 Read/Write during rendering - CXinc + Yinc (PPU.cs, MEM.cs)
- **Problem**: On $2007 read/write, VRAM address increment always used VramaddrIncrement (+1 or +32)
- **Correct behavior**: During rendering (visible + pre-render scanlines), $2007 access should trigger CXinc() + Yinc() (coarse X + Y increment), not the normal +1/+32
- **Fix**:
  - Added `Increment2007()` helper method to PPU.cs
  - Condition: `(ShowBG || ShowSpr) && (scanline < 240 || scanline == 261)` → CXinc + Yinc
  - All vram_addr increments in ppu_read_fun/ppu_write_fun in MEM.cs changed to use `Increment2007()`
- **Effect**: AccuracyCoin "$2007 Read w/ Rendering" (P16) FAIL → PASS

### 2. $2004 reads during rendering return $FF (PPU.cs)
- **Problem**: CPU reading $2004 during rendering returned normal OAM values
- **Correct behavior**:
  - Dots 1-64 (secondary OAM clear): $2004 read returns $FF
  - Dots 257-320 (sprite tile loading): $2004 read returns $FF
  - Dots 65-256 (sprite evaluation): $2004 read returns OAM evaluation latch (not implemented)
- **Fix**: Added scanline/dot condition checks to ppu_r_2004(), returning $FF for the above two ranges
- **Effect**: AccuracyCoin "$2004 Behavior" (P18) err=4 → err=8 (tests 1-7 pass, test 8 needs evaluation latch)

### 3. $2004 writes during rendering (PPU.cs)
- **Problem**: $2004 writes during rendering wrote to OAM normally and incremented OAMADDR by 1
- **Correct behavior**: Write does not modify OAM; OAMADDR increments by 4 and aligns to 4-byte boundary
- **Fix**: Added rendering condition to ppu_w_2004() (visible + pre-render scanlines), `spr_ram_add = (spr_ram_add + 4) & 0xFC`
- **Effect**: Combined with $2004 read fix, tests 6-7 now pass

## Test Results

- Legacy tests: 174/174 PASS (no regression)
- AccuracyCoin: 108/136 PASS (+1), 27 FAIL, 1 SKIP
- New PASS: $2007 Read w/ Rendering (P16)
- Improved but still FAIL: $2004 Behavior (P18, err=4→err=8)
