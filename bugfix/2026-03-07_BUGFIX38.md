# BUGFIX38: OAM DMA deferred execution (INC $4014 fix)

**日期**: 2026-03-07
**影響測試**: AccuracyCoin P18 $0480 (INC $4014) err=3 → PASS
**基線變化**: AC 112/136 → 113/136 PASS, blargg 174/174 不變

## 問題描述

`INC $4014` (RMW instruction) 對 $4014 產生兩次寫入:
1. Dummy write: 寫回原始值 → 觸發 OAM DMA (513+ cycles)
2. Real write: 寫入 incremented 值 → 觸發第二次 OAM DMA

真實 NES 硬體中，DMA 無法在 CPU write cycle 啟動，因此只有一次 DMA 執行（使用第二次寫入的 page number）。

## 修復內容

### PPU.cs
- `ppu_w_4014()` 改為僅設定 `oamDmaPending = true` 和 `oamDmaPendingPage = value`
- 新增 `oamDmaExecute()` 包含原有的 DMA 執行邏輯

### MEM.cs
- `Mem_r()` 和 `ZP_r()` 開頭檢查 `oamDmaPending`，若有則執行 DMA
- DMA 在下一個 read cycle 才啟動，匹配真實硬體行為

### APU.cs
- 新增 `oamDmaPending` 和 `oamDmaPendingPage` 欄位
- `initAPU()` 中初始化 pending flag

## 測試結果
- blargg 174/174 PASS (無回歸)
- AccuracyCoin 113/136 PASS (+1: INC $4014)
