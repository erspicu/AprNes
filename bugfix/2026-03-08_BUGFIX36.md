# BUGFIX36: OAM Corruption

## 日期: 2026-03-08

## 基線: 174 PASS (舊測試), 110/136 AccuracyCoin PASS (BUGFIX35 後)

## 修復內容

### OAM Corruption (PPU.cs) — AccuracyCoin $047B PASS
- **問題**: NES PPU 在 rendering 從 enabled 切換為 disabled 時，會根據當前 PPU dot 標記 OAM row 為 corrupted。下次 rendering 再 enable（或 frame start）時，OAM row 0 的前 8 bytes 會被複製到被標記的 row，導致 sprite 消失或重複。AprNes 完全沒有實作此行為。
- **正確行為** (參考 Mesen2 NesPpu.cpp):
  - Dots 0-63 (Secondary OAM clear): 每 2 dots 標記 1 個 row (`corruptOamRow[dot >> 1]`)
  - Dots 256-319 (Sprite tile fetch): 8-dot segments，前 3 dots 遞增 row，後 5 dots 標記下一 row
  - 標記後，下次 rendering enable 或 frame start 時，copy OAM[0:7] 到每個被標記的 row
- **修復**:
  - 新增 `bool[] corruptOamRow = new bool[32]` 和 `bool prevRenderingEnabled`
  - `SetOamCorruptionFlags()`: 根據 ppu_cycles_x 計算要標記的 row
  - `ProcessOamCorruption()`: 遍歷 32 rows，copy OAM[0:7] 到被標記的 row
  - `ppu_w_2001()`: 偵測 rendering enabled↔disabled 轉換，呼叫對應方法
  - Frame start (scanline 262→0): 若 rendering enabled，呼叫 `ProcessOamCorruption()`
