# BUGFIX47: AccuracyCoin P19 Sprites On Scanline 0

**Date**: 2026-03-08
**Baseline**: 119/136 → **120/136** (+1)
**Fixed Test**: P19 Test 5 (Sprites On Scanline 0) err=2 → PASS

## Root Cause

P19 Test 5 verifies that sprites can correctly appear on scanline 0.

On real NES PPU, during the pre-render line (261) dots 257-320, sprite tile data is fetched
from secondary OAM using `(261 & 255) = 5` as the effective scanline for in-range checks.
Secondary OAM retains the evaluation results from the previous visible scanline (239).
If secondary OAM contains a sprite whose Y position places it within range of scanline 5,
that sprite's tile data is loaded into shift registers and persists through scanline 0.

The previous implementation was missing:
1. Secondary OAM buffer and per-dot sprite evaluation FSM
2. Pre-render line sprite data calculation
3. Using pre-render sprite data for sprite 0 hit detection on scanline 0

## Fix

### PPU.cs — Secondary OAM and Per-dot Evaluation

Added secondary OAM (32 bytes) and a complete per-dot sprite evaluation FSM:

- **Dots 1-64**: Clear secondary OAM (write $FF, one byte every 2 dots)
- **Dot 65**: Initialize evaluation state (SpriteEvalInit)
- **Dots 65-256**: Per-dot evaluation (SpriteEvalTick) — odd dot reads primary OAM, even dot writes/checks secondary OAM
- **Dot 256**: Record evalSpriteCount and evalSprite0Visible (SpriteEvalEnd)
- **Pre-render dot 257**: PrecomputePreRenderSprites() — checks whether secondary OAM contains any sprite in range of scanline 5

### PPU.cs — ppu_r_2004()

Changed to return `oamCopyBuffer` (the last byte read by the evaluation FSM), replacing
the previous approximation. Dots 257-320 return the corresponding secondary OAM byte.

### PPU.cs — PrecomputeSprite0Line()

On scanline 0, if the primary OAM sprite is out of range, use `prerender_sprite0` data.

## Notes

Sprite overflow flag is still set by PrecomputeOverflow() at the precise cycle.
Duplicate isSpriteOverflow assignments were removed from the FSM to avoid blargg regression.

- blargg: 174/174 (no regression)
- AccuracyCoin: 120/136 (+1, P19 Test 5 PASS)
