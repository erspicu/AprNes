# BUGFIX33: AccuracyCoin Phase 1 - Controller strobe, OAMADDR reset, S0H rendering flags

## 日期: 2026-03-06

## 基線: 171 PASS (舊測試), 106/136 AccuracyCoin PASS

## 修復內容

### 1. Controller Strobing bit 0 check (JoyPad.cs)
- **問題**: $4016 strobe 只檢查 `P1_LastWrite == 1`，但 NES 硬體檢查 bit 0
- **修復**: 改為 `(P1_LastWrite & 1) == 1 && (val & 1) == 0`
- **效果**: AccuracyCoin Controller Strobing 從 err=2 改善至 err=4 (3/4 subtests pass)
- Test 4 (put/get cycle parity) 仍 FAIL，需 CPU bus cycle 奇偶追蹤

### 2. OAMADDR reset during rendering (PPU.cs)
- **問題**: 渲染期間未重置 spr_ram_add (OAMADDR)，導致 OAM DMA 後地址偏移
- **根因**: NES 硬體在每條 scanline 的 dot 257 重置 OAMADDR 為 0
- **修復**: ppu_rendering_tick() dot 257 加入 `spr_ram_add = 0`
- **影響範圍**:
  - Rendering Flag Behavior: FAIL → PASS (BG shift regs 在 sprites-only 模式下正確運作)
  - $2004 behavior: err=1 → err=4 (基本 OAM 寫入+DMA 測試通過)
  - 修復了多個 sprite zero 相關測試的前提條件

### 3. Sprite 0 hit detection flags (PPU.cs)
- **問題**: PrecomputeSprite0Line 要求兩個旗標都啟用才預計算 (||)
- **修復**: 改為任一旗標啟用即預計算 (&&)，但 per-pixel 偵測改為要求兩個旗標
  - PrecomputeSprite0Line: `!ShowBG || !ShowSpr` → `!ShowBG && !ShowSpr`
  - Per-pixel S0H: `renderingEnabled` → `ShowBackGround && ShowSprites`
- **原因**: 資料可在 sprites-only 模式下預計算，但 hit 只在兩者都開啟時觸發

## 測試結果

- 舊測試: 174/174 PASS (無回歸)
- AccuracyCoin: 107/136 PASS (+1), 28 FAIL, 1 SKIP
