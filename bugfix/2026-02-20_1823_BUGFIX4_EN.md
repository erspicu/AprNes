# BUGFIX4: Headless Test Runner + CPU/PPU/APU Accuracy Fixes

Date: 2026-02-20

## I. Headless Console Test Runner (New Feature)

Added headless mode: ROMs can be loaded automatically via command-line parameters, tests run, screenshots taken, and results logged.

### New Files
- **TestRunner.cs** — core test logic
- **run_tests.ps1** — PowerShell batch test script
- **test_output/report.md** — test result report

### Modified Files
- **Program.cs** — dual-mode entry point: with args → headless, without args → GUI
- **AprNes.csproj** — add TestRunner.cs

### Usage
```
AprNes.exe --rom test.nes --wait-result --max-wait 30 --screenshot out.png --log results.log
powershell -ExecutionPolicy Bypass -File run_tests.ps1
```

### Supported Parameters
| Parameter | Description |
|-----------|-------------|
| `--rom <path>` | ROM file path |
| `--wait-result` | Wait for blargg $6000 status completion |
| `--max-wait <sec>` | Maximum wait time in seconds (default 30) |
| `--time <sec>` | Stop after running for specified seconds |
| `--screenshot <path>` | Screenshot output path |
| `--log <path>` | Result log path |
| `--debug-log <path>` | Debug log path |
| `--debug-max <n>` | Maximum debug log lines |

## II. Mapper WRAM Fix

The blargg test framework writes results to $6000-$7FFF. The following mappers had empty RAM handlers; fixed to read/write NES_MEM:

- **Mapper000.cs** / **Mapper002.cs** / **Mapper003.cs** / **Mapper005.cs** / **Mapper011.cs** / **Mapper066.cs** / **Mapper071.cs**

```csharp
// Before:
public void MapperW_RAM(ushort address, byte value) { }
public byte MapperR_RAM(ushort address) { return 0; }
// After:
public void MapperW_RAM(ushort address, byte value) { NesCore.NES_MEM[address] = value; }
public byte MapperR_RAM(ushort address) { return NesCore.NES_MEM[address]; }
```

## V. CPU Dummy Read Fixes (CPU.cs)

### Bug 4: Read instructions missing page-cross dummy read (25 opcodes)

On the 6502 CPU, indexed addressing with a page cross first reads a "wrong page" address (triggering I/O side effects), then reads the correct address. Previously only 1 cycle was added without an actual read.

Fixed opcodes:
- **abs,X** (6): ADC(0x7D), AND(0x3D), CMP(0xDD), EOR(0x5D), LDY(0xBC), SBC(0xFD)
- **abs,Y** (7): ADC(0x79), AND(0x39), CMP(0xD9), EOR(0x59), LDA(0xB9), LDX(0xBE), SBC(0xF9)
- **(ind),Y** (6): ADC(0x71), AND(0x31), CMP(0xD1), EOR(0x51), ORA(0x11), SBC(0xF1)

Fix pattern:
```csharp
// Before:
byte1 = Mem_r(ushort2);  // read correct address directly
if ((ushort1 & 0xff00) != (ushort2 & 0xff00)) cpu_cycles++;

// After:
if ((ushort1 & 0xff00) != (ushort2 & 0xff00))
{
    Mem_r((ushort)((ushort1 & 0xFF00) | (ushort2 & 0x00FF))); // dummy read wrong page
    cpu_cycles++;
}
byte1 = Mem_r(ushort2);  // correct read
```

### Bug 5: STA abs,Y (0x99) missing dummy read

Store instructions always perform a dummy read (regardless of page cross); the original was completely missing it.

```csharp
// Before:
Mem_w((ushort)((Mem_r(r_PC++) | (Mem_r(r_PC++) << 8)) + r_Y), r_A);

// After:
ushort1 = (ushort)(Mem_r(r_PC++) | (Mem_r(r_PC++) << 8));
ushort2 = (ushort)(ushort1 + r_Y);
Mem_r((ushort)((ushort1 & 0xFF00) | (ushort2 & 0x00FF))); // dummy read (always)
Mem_w(ushort2, r_A);
```

### Bug 6: RMW abs,X missing wrong-page dummy read (5 opcodes)

RMW (Read-Modify-Write) instructions with abs,X addressing always read the wrong-page address first.

Fixed opcodes: ASL(0x1E), LSR(0x5E), ROR(0x7E), DEC(0xDE), INC(0xFE)

```csharp
// Before:
ushort2 = (ushort)((Mem_r(r_PC++) | (Mem_r(r_PC++) << 8)) + r_X);

// After:
ushort1 = (ushort)(Mem_r(r_PC++) | (Mem_r(r_PC++) << 8));
ushort2 = (ushort)(ushort1 + r_X);
Mem_r((ushort)((ushort1 & 0xFF00) | (ushort2 & 0x00FF))); // dummy read wrong page
```

### Bug 7: Unofficial NOP abs,X missing page-cross cycle (6 opcodes)

0x1C, 0x3C, 0x5C, 0x7C, 0xDC, 0xFC — originally only did `r_PC += 2` with no page-cross detection.

```csharp
// Before:
r_PC += 2;

// After:
ushort1 = (ushort)(Mem_r(r_PC++) | (Mem_r(r_PC++) << 8));
ushort2 = (ushort)(ushort1 + r_X);
if ((ushort1 & 0xff00) != (ushort2 & 0xff00))
{
    Mem_r((ushort)((ushort1 & 0xFF00) | (ushort2 & 0x00FF)));
    cpu_cycles++;
}
Mem_r(ushort2); // actual read (discarded)
```

### Bug 8: NOP abs (0x0C) not reading target address

Changed to actually read and discard.

### Bug 9: 0xE2 cycle table error

cycle_table[0xE2] corrected from 3 to 2 (NOP immediate, 2 bytes 2 cycles).
