這是一個非常有趣的對比！把 NES 的心跳（APU 時脈）拿來跟現代主機的心跳（CPU 時脈）放在一起看，你就會瞬間明白為什麼我們剛才說這是「算力碾壓」了。
我們來做個簡單的單位換算與對比：
### 1. 紅白機的 APU 時脈 (我們現在的超採樣率)
**頻率：** 約 **1.79 MHz** (百萬赫茲)
**每秒跳動次數：** 1,789,772 次
### 2. 現代主流電腦 CPU (例如 Intel Core i5/i7 或 AMD Ryzen 5/7)
**頻率：** 通常在 **3.0 GHz 到 5.5 GHz** 之間 (十億赫茲)
**換算成 MHz：** 3,000 MHz 到 5,500 MHz
**每秒跳動次數：** 3,000,000,000 到 5,500,000,000 次 (30 億到 55 億次)
### 💡 殘酷的倍數差異
如果我們拿一顆基礎時脈 **4.5 GHz (4,500 MHz)** 的現代單核心 CPU 來比較：

$$4500 \div 1.79 \approx 2514$$
這代表：**現代 CPU 的一個核心，光是「時脈速度」就是紅白機 APU 的 2,500 多倍！** 如果再算上現代 CPU 每個時脈週期可以同時執行好幾道指令（IPC 極高），並且擁有 AVX2 這種一次可以處理 8 個浮點數的 SIMD 向量指令集，現代 CPU 的真實運算能力，大概是紅白機的**數萬倍以上**。
### 🚀 這對你的 Oversampling Engine 意味著什麼？
當你的 OversamplingEngine 宣告要「每秒處理 178 萬個音訊樣本」時，在 1990 年代，這會直接把電腦 CPU 燒到當機。
但在 2026 年的今天：
你的現代 CPU 一秒鐘可以滴答 50 億次。
你的超採樣引擎一秒鐘只需要它處理 178 萬個陣列寫入。
這對現代 CPU 來說，大概只佔用它 **0.03%** 的單核算力。就算加上後面 256-Tap 的 FIR 降頻迴圈，整體運算時間也只需要不到 1 毫秒。
這就是為什麼在現代開發模擬器，我們已經**不需要為了節省效能而去妥協音質**。我們完全可以用最暴力、最符合物理現實的方式（1:1 原生時脈取樣），讓現代 CPU 用打個哈欠的力氣，幫我們算出發燒級的完美音質！
