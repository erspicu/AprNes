### 技術亮點

#### BUGFIX53: DMC Load DMA Countdown (132→133)

DMC Load DMA countdown 改為 TriCNES 風格的 GET-only decrement。
原本每 cycle 遞減，改為僅在 GET cycle 遞減，修正 DMA + $2002 Read 的 phantom read timing。

#### BUGFIX54: DMC DMA Bus Conflicts (133→134)

DMC DMA 讀取 APU 暫存器（$4000-$4017）時的 bus conflict 重寫。
新增 deferred $4015 status update 機制：$4015 寫入的 enable/disable 效果延遲 3-4 cycles 生效，
避免 DMA 期間的 status 變更干擾正在進行的 DMA。

#### BUGFIX55: Explicit DMA Abort (134→135)

當 DMC DMA 正在進行時寫 $4015=#$00 (disable)，需要正確計算 deferred status delay。
覆蓋 2-cycle fire window：
- `dmctimer == dmcrate`：timer 剛 fire（delay=4）
- `dmctimer == 1`：timer 即將 fire（delay=5）

Normal delay 改為 parity-dependent：`getCycle ? 4 : 3`。

#### BUGFIX56: Implicit DMA Abort (135→136)

最後一個失敗項。實作「幽靈」1-cycle phantom DMA 機制：

1. **偵測條件**：`(dmctimer == 8 && !getCycle) || (dmctimer == 9 && getCycle)`
   — 經驗驗證存在 +3 position offset（pending→active 轉換延遲）
2. **Post-halt 取消**：halt cycle 後立即取消 DMA（僅 `dmcsamplesleft == 0` 時）
3. **Write cycle 取消**：CpuWrite 中 implicit abort active + halt pending → 直接取消 DMA
