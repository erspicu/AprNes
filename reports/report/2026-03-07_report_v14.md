### 技術亮點

#### BUGFIX45: $2002 Flag Clear Timing

最後一項修復解決了一個之前認為「不可能不回歸」的問題。

**問題**：AccuracyCoin P18 測試 `$2002 flag clear timing` 要求 sprite flags（sprite 0 hit、sprite overflow）比 VBL flag 更早清除。

**之前失敗的嘗試**：
- Dot 2→3 VBL stagger：回歸 P17 VBlank End (-2)
- Dot 1 統一清除：無改善

**突破**：分析 AccuracyCoin 原始碼發現 P17 VBlank End 只檢查 VBL flag（bit 7），不檢查 sprite flags。因此可以安全地將 sprite flags 清除提前到 dot 1，VBL 維持 dot 2。

```csharp
// Before: all flags at dot 2
if (scanline == 261 && ppu_cycles_x == 2)
    isVblank = isSprite0hit = isSpriteOverflow = false;

// After: sprite flags at dot 1, VBL at dot 2
if (scanline == 261 && ppu_cycles_x == 1)
    isSprite0hit = isSpriteOverflow = false;
if (scanline == 261 && ppu_cycles_x == 2)
    isVblank = false;
```

這模擬了真實 RP2A03G 的 M2 duty cycle effect：sprite flags 在 M2 fall 時讀取（較晚），VBL 在 M2 rise 時讀取（較早），使得 sprite flags 看起來比 VBL 早 ~2 PPU dots 清除。

### 剩餘問題分析

17 FAIL + 1 SKIP 全部屬於高難度，主要分為三個根因：

| 根因 | 數量 | 說明 |
|------|------|------|
| DMA sub-cycle 精度 | 12 | P10(5) + P13(6) + P20(1)，需 Master Clock |
| DMC/APU bus 互動 | 3 | P14 DMC/APU Reg/Controller Strobe |
| PPU per-dot 精度 | 3 | P19 BG serial/sprites SL0/$2004 stress |
| DMC DMA 漂移 | 1 | P12 IFlagLatency Test E hang |

**結論**：在不變更 timing 架構的前提下，已無明顯可修復的項目。進一步提升需要 Master Clock 驅動架構重構。
