### 2. AprNesAOT Full UI Port (Equivalent to WinForms Features)

**Goal**: Fully port all "basic operations" from the original WinForms UI to a pure Win32 P/Invoke implementation.

**Approach**:

#### Menu (4 top-level menus)
- **File**: Open ROM, Soft Reset, Hard Reset, Exit
- **Options**: Sound toggle (checkmark), Volume submenu (10/30/50/70/90/100% Radio), Limit FPS (checkmark)
- **Language**: Dynamically generated from LangINI, with current language checked
- **Help**: ROM Info, About

#### Dynamic Behavior
- While no ROM is loaded, "Soft Reset", "Hard Reset", and "ROM Info" are grayed out (`EnableMenuItem`)
- Automatically enabled after a ROM is loaded

#### FPS Display
- `SetTimer` WM_TIMER updates the title bar every second: `AprNes AOT  [rom_name.nes]  FPS: 60`

#### Settings Read/Write
- Reads and writes `AprNes.ini` (key bindings, sound, volume, FPS limit, language)
- Format is identical to the original WinForms version (`key=value` plain text)

#### Reset
- **Soft Reset**: `_event.Reset()` → `NesCore.SoftReset()` → `_event.Set()`
- **Hard Reset**: stop emulation thread → `NesCore.init(_lastRomBytes)` → restart thread

#### New Public Properties in NesCore (`Main.cs`)

```csharp
static public int  RomMapper     => mapper;
static public int  RomPrgCount   => PRG_ROM_count;
static public int  RomChrCount   => CHR_ROM_count;
static public bool RomHorizMirror => (ROM_Control_1 & 1) == 0;
static public bool LimitFPS      = false;
```

### 3. Bugfix: SetBkMode / SetTextColor Wrong DLL

**Symptom**: Application crashes a few seconds after launch:
```
EntryPointNotFoundException: 'SetBkMode' in DLL 'user32.dll'
```

**Cause**: `SetBkMode` / `SetTextColor` belong to GDI (`gdi32.dll`), but were incorrectly declared under `user32.dll`.

**Fix**:
```csharp
// before
[DllImport("user32.dll")] static extern nint SetBkMode(...);
// after
[DllImport("gdi32.dll")]  static extern nint SetBkMode(...);
```
