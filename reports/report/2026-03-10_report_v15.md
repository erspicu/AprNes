### 技術亮點

#### Per-cycle CPU Rewrite (122→126, +4)

將 CPU 從「每指令一次性執行所有 cycles」改為「每 cycle 獨立步進」模型。
每個 CPU cycle 都是獨立的 `StartCpuCycle → bus op → EndCpuCycle`，
DMA 可在任意 cycle 邊界插入，精確模擬 stolen cycle 行為。

**修復的測試**:
- P12 IFlagLatency：Test E 不再 hang（DMC DMA cycle drift 消除）
- P20 Instruction Timing + Implied Dummy Reads：DMA 前置條件通過
- P13 DMA + $4015 Read / DMC DMA + OAM DMA：DMA timing 對齊

#### SH* Unofficial Opcodes Fix (126→131, +5)

SHA/SHX/SHY/SHS 指令的 DMA bus conflict 行為正確實作。
當 DMA 發生在 SH* write cycle 前，value = A & X（消除 H 的 AND masking）。
P10 全 6 項測試 PASS。

#### DMC DMA Cooldown (131→132, +1)

參考 TriCNES 的 `CannotRunDMCDMARightNow` 機制，
在 DMC DMA 完成後設置 cooldown 期間，防止連續 DMC DMA 觸發。

### 剩餘問題

4 FAIL / 0 SKIP，全部在 P13 DMA Tests：

| 測試 | err | 分析 |
|------|-----|------|
| DMA + $2002 Read | 2 | phantom read 碰 $2002 時序 |
| DMC DMA Bus Conflicts | 2 | bus conflict 合併時序 |
| Explicit DMA Abort | 2 | DMA 中止 stolen cycle 數 |
| Implicit DMA Abort | 2 | 隱式 DMA 中止 stolen cycle 數 |

共用根因：DMA sub-cycle 精度不足。深入調查確認需要三個修復同時到位：
1. **Asymmetric DMA split** — CpuRead 中 ProcessPendingDma 移到 StartCpuCycle 之前（Write 不變）
2. **Abort mechanism** — 延遲 $4015 disable + explicit/implicit abort 的 stolen cycle 計數
3. **Bus conflict 邏輯** — DMC DMA 讀 APU 暫存器時的 bus-OR 行為

已驗證: 延遲 $4015 disable 單獨實作無效；簡單 reorder 破壞 6 blargg；reorder+parity flip 全壞。
