### Technical Highlights

#### BUGFIX53: DMC Load DMA Countdown (132→133)

The DMC Load DMA countdown was changed to a TriCNES-style GET-only decrement.
Previously decremented every cycle; now decrements only on GET cycles, fixing the phantom read timing for DMA + $2002 Read.

#### BUGFIX54: DMC DMA Bus Conflicts (133→134)

Rewrote bus conflict handling when DMC DMA reads APU registers ($4000-$4017).
Added a deferred $4015 status update mechanism: enable/disable effects from $4015 writes are delayed 3-4 cycles before taking effect,
preventing in-flight status changes from disrupting an ongoing DMA.

#### BUGFIX55: Explicit DMA Abort (134→135)

When $4015=#$00 (disable) is written while a DMC DMA is in progress, the deferred status delay must be computed correctly.
Covers the 2-cycle fire window:
- `dmctimer == dmcrate`: timer just fired (delay=4)
- `dmctimer == 1`: timer about to fire (delay=5)

Normal delay changed to parity-dependent: `getCycle ? 4 : 3`.

#### BUGFIX56: Implicit DMA Abort (135→136)

The final failing item. Implemented a "ghost" 1-cycle phantom DMA mechanism:

1. **Detection condition**: `(dmctimer == 8 && !getCycle) || (dmctimer == 9 && getCycle)`
   — empirically verified with a +3 position offset (pending→active transition delay)
2. **Post-halt cancellation**: cancel the DMA immediately after the halt cycle (only when `dmcsamplesleft == 0`)
3. **Write cycle cancellation**: in CpuWrite, if implicit abort is active and halt is pending → cancel the DMA directly
