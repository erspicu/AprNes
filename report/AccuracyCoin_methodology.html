1. About AccuracyCoin

AccuracyCoin is a comprehensive NES accuracy test ROM that goes beyond the standard blargg test suites. It contains 136 individual tests plus 5 DRAW-only information pages, covering CPU instructions, unofficial opcodes, interrupt timing, DMA behavior, APU registers, PPU rendering, and edge-case hardware behaviors.

  • ROM path: nes-test-roms-master/AccuracyCoin-main/AccuracyCoin.nes
  • Mapper: 0 (NROM)
  • Author: Community project with assembly source available at AccuracyCoin.asm
  • Scope: Tests behaviors that most emulators get wrong — sub-cycle DMA timing, M2 duty cycle effects, OAM evaluation internals, unstable unofficial opcodes, etc.

run_ac_test.sh — Single Page/Item Testing

The primary development tool for debugging individual tests:

# Run all tests on a page
bash run_ac_test.sh 18

# Run a single test item (1-based index)
bash run_ac_test.sh 14 7          # P14, item 7 (APU Register Activation)

# Skip an item that hangs, run the rest
bash run_ac_test.sh 12 --skip 1   # P12, skip IFlagLatency

# Skip build, increase timeout
bash run_ac_test.sh 13 --no-build --extra-time 10

The script automatically:

  • Builds the emulator (unless --no-build)
  • Calculates input timing for page navigation (Right key presses)
  • Sends A to start the test, waits for completion
  • Takes a screenshot and dumps the result hex data

run_tests_AccuracyCoin_report.sh — Full Report

# Full run (build + all 20 pages + screenshots + HTML report)
bash run_tests_AccuracyCoin_report.sh

# Skip build step
bash run_tests_AccuracyCoin_report.sh --no-build

# Skip screenshots (faster, data only)
bash run_tests_AccuracyCoin_report.sh --no-screenshots

Output:

  • HTML report: report/AccuracyCoin_report.html
  • Page screenshots: report/screenshots-ac/ac_page_XX.png
  • Intermediate hex data: temp/ac_results/page_XX.hex

Why Per-Page Testing?

AccuracyCoin's Start button runs all 136 tests sequentially, but Page 12 item 1 (IRQ Flag Latency Test E) hangs indefinitely due to DMC DMA timing drift. This blocks all subsequent tests. The per-page architecture solves this by running each page in an independent emulator instance.

Per-Page Flow

  1. Independent boot: Each page launches a fresh emulator instance (20 launches total)
  2. Navigation: Right key pressed (N-1) times to reach Page N from Page 1
  3. P12 special handling: Navigate to item 1 → press B (mark SKIP) → return to title → press A
  4. P15 special handling: DRAW-only page, screenshots captured but no PASS/FAIL data
  5. Result collection: --dump-ac-results outputs hex data for memory range $0400-$05FF
  6. Merge: All 20 pages' hex data are merged (non-zero values overwrite zeroes)

Timing Calculation

ROM boot wait:     3.0 seconds
Nav key interval:  0.5 seconds per press
Post-nav settle:   1.0 seconds
Test wait time:    8-30 seconds (varies by page)

Page N total time ≈ 3.0 + (N-1) × 0.5 + 1.0 + test_wait

6. Result Encoding

Memory Layout

AccuracyCoin stores each test result as 1 byte at a fixed address in the $0400–$05FF range:

ValueMeaningDecoding
0x01PASSbit 0 = 1
0x02FAIL error 0(0 << 2) | 0x02
0x06FAIL error 1(1 << 2) | 0x02
0x0AFAIL error 2(2 << 2) | 0x02
0x1EFAIL error 7(7 << 2) | 0x02
0xFFSKIPMarked by B button
0x00Not executed—

General formula: error_code = result_byte >> 2 (when bit 1 is set)

Error Code Meanings

Each test defines its own error codes. Common patterns for unofficial opcode tests:

  • 1: Target address incorrect
  • 2: A register value incorrect
  • 3: X register value incorrect
  • 4: Y register value incorrect
  • 5: CPU flags incorrect
  • 6: Stack pointer incorrect
  • 7+: DMA bus conflict / timing-dependent behavior

8. Known Limitations

Hanging Test

TestPageIssueRoot Cause
IRQ Flag Latency (Test E)P12, item 1Infinite loopDMC DMA cumulative timing drift (~12 cycles). Requires Master Clock scheduler.

Remaining FAIL Categories

Root CauseCountDescription
DMA sub-cycle precision12P10 SH* (5) + P13 DMA (6) + P20 dummy reads (1). All share the same DMADMASync precondition failure.
DMC/APU bus interaction3P14: DMC channel, APU register activation during DMA, controller strobe PUT/GET parity.
PPU per-dot precision3P19: BG shift register reload timing, secondary OAM persistence, OAM evaluation state machine.

Highest ROI Fix Target

Fixing the DMADMASync_PreTest precondition (sub-cycle accurate DMC DMA open bus timing) could unlock +6 to +12 tests at once, covering P13, P10, and P20.
