# 2026-03-02 開發日誌（DirectInput 重構 + 搖桿設定修復）

## 本次改動摘要

---

### 1. WinMM → DirectInput8 + XInput 重構

**目標**：WinMM `joyGetPos` API 過於老舊，不支援 XInput 手把、D-pad POV hat，且已不建議使用。

**做法**：
- 新增 `AprNes/tool/DirectInputHelper.cs`：以 raw vtable dispatch 實作 DirectInput8 COM 操作
- 重寫 `AprNes/tool/joystick.cs`：DirectInput（non-XInput）+ XInput 雙路徑
- 移除 `NativeAPIShare.cs` 中所有 WinMM P/Invoke 與結構體

**遭遇問題**：`[ComImport]` RCW 在傳入 managed delegate 給 `EnumDevices` 時引發 `AccessViolationException`

**解決**：改用 raw vtable 手動 dispatch + `Marshal.GetFunctionPointerForDelegate` 傳入 callback IntPtr，完全繞過 CLR 的 COM marshal 機制。

---

### 2. 設定介面修復 — 非 Button 類型輸入警告

**問題**：設定介面在未按任何鍵時不斷彈出「非Button類型輸入!」MessageBox。

**原因**：DirectInput 軸靜止時有硬體雜訊事件，且設定介面無事件類型篩選。

**修復**：
- `AXIS_NOISE=256`：靠近中心 ±256 的軸值歸一化為 32767
- `ExpectedJoyInputType()`：A/B/Start/Select → type 1（Button only）；UP/DOWN/LEFT/RIGHT → type 2（any）
- configure polling 依 expected type 靜默過濾不符事件

---

### 3. 設定介面修復 — 方向鍵欄位無法設定（防禦性改進）

**問題**：`Setup_JoyPad_define()` 使用 `.Focused` 判斷目前設定欄位，在跨執行緒 Invoke + modal ShowDialog 的組合下不可靠。

**修復**：以 `_activeJoyControl` 欄位 + `Click`/`Enter` 事件明確追蹤使用者點擊的欄位，不依賴 Windows 焦點狀態。

---

### 4. 方向鍵設定 Root Cause 修復 — 軸值不精確

**問題**：JoyTest 診斷確認 D-pad D 模式有 `event_type=0 AXIS` 事件，但方向鍵設定和遊戲仍無反應。

**根本原因**：`JoyPadWayName()` 要求 axis value 精確等於 `0` 或 `65535`。若 `DIPROP_RANGE` 設定失敗或手把回傳偏差值（如 `1`、`65434`），函式返回空字串，設定和 keymap 查詢均失敗。

**修復**：新增 `SnapAxisDigital(v)`：
- `v < 16384` → `0`
- `v > 49151` → `65535`
- 其他 → `32767`

套用到所有 DirectInput / XInput 軸值，保證 `JoyPadWayName` 始終收到精確值。

---

### 5. JoyTest 診斷工具

新增 `tools/JoyTest.cs` 獨立 WinForms 診斷工具，用於確認 DirectInput / XInput 裝置事件內容（event_type、軸值、按鈕、POV、raw 值）。
