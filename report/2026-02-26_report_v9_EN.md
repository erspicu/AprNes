### 2. NesCore System-Layer Refactoring (per MD/NesCore_refactor_proposal.md)

**Goal**: Make NesCore a pure emulation logic core; system-layer concerns (audio output, FPS throttling, error display) become the caller's responsibility.

#### 🔴 High Priority: Move APU Audio Output Out of NesCore

**Before**: `APU.cs` contained both audio emulation logic and WinMM WaveOut system calls, hard-coupling NesCore to Windows.

**Approach**:
- Created `AprNes/WaveOutPlayer.cs`:
  - Contains WaveOut P/Invoke (WAVEFORMATEX, WAVEHDR, waveOutOpen/Write/Close, etc.)
  - Subscribes to `NesCore.AudioSampleReady`; receives PCM samples and feeds them into the WaveOut buffer for playback
  - `OpenAudio()` / `CloseAudio()` serve as the external interface
  - `timeBeginPeriod(1)` / `timeEndPeriod(1)` managed inside OpenAudio/CloseAudio
- `APU.cs` changes:
  - Removed all WaveOut P/Invoke declarations, structs, and buffer fields (`_hWaveOut`, `_audioBufs`, etc.)
  - Removed `openAudio()`, `closeAudio()`, `submitBuffer()` methods
  - Added `static public Action<short> AudioSampleReady`
  - `generateSample()` now calls `AudioSampleReady?.Invoke((short)clamped)`
  - Removed `using System.Runtime.InteropServices` and `using System.Threading`

#### 🔴 High Priority: Replace ShowError with a Delegate, Remove Windows.Forms Dependency

**Before**: `Main.cs` had `using System.Windows.Forms`; `ShowError()` called `MessageBox.Show()` directly.

**Approach**:
- Added `static public Action<string> OnError`
- `ShowError()` changed to `OnError?.Invoke(msg)`, no longer branching on HeadlessMode
- Removed `using System.Windows.Forms`
- UI side sets: `NesCore.OnError = msg => MessageBox.Show(msg);`
- TestRunner sets: `NesCore.OnError = msg => Console.Error.WriteLine("ERROR: " + msg);`

#### 🟡 Medium Priority: Move FPS Throttling Out of PPU

**Before**: `PPU.cs` performed Hybrid Sleep rate-limiting after scanline 240 was rendered and owned the Stopwatch.

**Approach**:
- Removed `Stopwatch`, `_fpsDeadline`, `NES_FRAME_SECONDS`, and `LimitFPS` fields from `PPU.cs`
- Removed the FPS throttle block (`if (LimitFPS) { ... }`)
- Removed `using System.Diagnostics` and `using System.Threading`
- The `VideoOutputDeal` handler in `AprNesUI.cs` takes over FPS throttling:
  - Added `_fpsStopWatch`, `_fpsDeadline`, `NES_FRAME_SECONDS` as UI fields
  - Executes Hybrid Sleep on each `VideoOutput` event
  - Resets deadline and restarts stopwatch on ROM load / Hard Reset

#### 🟢 Low Priority: Move timeBeginPeriod to the Caller

`timeBeginPeriod(1)` was previously called at the start of `NesCore.run()`. It has been moved into `WaveOutPlayer.OpenAudio()` / `WaveOutPlayer.CloseAudio()`; `run()` no longer holds this system call.
