1. 關於 AccuracyCoin

AccuracyCoin 是一個全面的 NES 精確度測試 ROM，比標準的 blargg 測試套件更深入。包含 136 個獨立測試及 5 個 DRAW 資訊頁，涵蓋 CPU 指令、非官方指令、中斷時序、DMA 行為、APU 暫存器、PPU 渲染以及硬體邊緣行為。

  • ROM 路徑：nes-test-roms-master/AccuracyCoin-main/AccuracyCoin.nes
  • Mapper：0 (NROM)
  • 組語原始碼：AccuracyCoin.asm（完整可查閱）
  • 測試範圍：測試大多數模擬器會做錯的行為 — sub-cycle DMA 時序、M2 duty cycle 效應、OAM evaluation 內部狀態、不穩定的非官方指令等

run_ac_test.sh — 單頁/單項測試

開發除錯的主要工具：

# 執行某頁全部測試
bash run_ac_test.sh 18

# 執行單項測試（1-based 索引）
bash run_ac_test.sh 14 7          # P14 第 7 項（APU Register Activation）

# 跳過會卡住的項目，執行其餘
bash run_ac_test.sh 12 --skip 1   # P12 跳過 IFlagLatency

# 跳過編譯、增加超時時間
bash run_ac_test.sh 13 --no-build --extra-time 10

腳本自動處理：

  • 編譯模擬器（可用 --no-build 跳過）
  • 計算頁面導覽的輸入時序（右鍵按壓次數與間隔）
  • 送出 A 鍵開始測試，等待完成
  • 擷取截圖並傾印結果 hex 資料

為何採用分頁測試？

AccuracyCoin 的 Start 鍵可連續執行全部 136 項測試，但 Page 12 第 1 項（IRQ Flag Latency Test E）會無限掛住，原因是 DMC DMA 時序漂移。這會導致後續所有測試無法執行。分頁架構透過每頁獨立啟動模擬器來解決此問題。

逐頁流程

  1. 獨立啟動：每頁各啟動一次模擬器（共 20 次），互不干擾
  2. 導覽：從 Page 1 按右鍵 (N-1) 次到達 Page N
  3. P12 特殊處理：導覽至第 1 項 → 按 B（標記 SKIP）→ 返回標題 → 按 A
  4. P15 特殊處理：DRAW 頁面，僅擷取截圖，無 PASS/FAIL 資料
  5. 結果收集：--dump-ac-results 輸出 $0400-$05FF 記憶體區域的 hex 資料
  6. 合併：20 頁的 hex 資料合併（非零值覆蓋零值），產生完整結果

時序計算

ROM 啟動等待：   3.0 秒
導覽按鍵間隔：   每次 0.5 秒
導覽後穩定：     1.0 秒
測試等待時間：   8-30 秒（依頁面而異）

Page N 總時間 ≈ 3.0 + (N-1) × 0.5 + 1.0 + 測試等待

6. 結果編碼

記憶體配置

AccuracyCoin 將每項測試結果以 1 byte 存放在 $0400–$05FF 範圍的固定位址：

值意義解碼方式
0x01PASSbit 0 = 1
0x02FAIL 錯誤碼 0(0 << 2) | 0x02
0x06FAIL 錯誤碼 1(1 << 2) | 0x02
0x0AFAIL 錯誤碼 2(2 << 2) | 0x02
0x1EFAIL 錯誤碼 7(7 << 2) | 0x02
0xFFSKIP被 B 鍵標記跳過
0x00尚未執行—

通用公式：error_code = result_byte >> 2（當 bit 1 = 1 時）

錯誤碼含義

每項測試各自定義錯誤碼。非官方指令測試的常見模式：

  • 1: 目標位址錯誤
  • 2: A 暫存器值錯誤
  • 3: X 暫存器值錯誤
  • 4: Y 暫存器值錯誤
  • 5: CPU flags 錯誤
  • 6: Stack pointer 錯誤
  • 7+: DMA bus conflict / 時序相關行為

卡住的測試

測試分頁問題根因
IRQ Flag Latency (Test E)P12 第 1 項無限迴圈DMC DMA 累積時序漂移（約 12 cycles），需要 Master Clock scheduler

剩餘 FAIL 分類

根因數量說明
DMA sub-cycle 精度12P10 SH*(5) + P13 DMA(6) + P20 dummy reads(1)。共用 DMADMASync 前置條件失敗。
DMC/APU bus 互動3P14：DMC 通道、DMA 期間 APU 暫存器啟用、controller strobe PUT/GET parity
PPU per-dot 精度3P19：BG shift register reload 時序、secondary OAM 持久化、OAM evaluation state machine

最高投資報酬率修復方向

修好 DMADMASync_PreTest 前置條件（sub-cycle 精確的 DMC DMA open bus timing）可能一次解鎖 +6 至 +12 項測試，涵蓋 P13、P10 和 P20。這需要 Master Clock 驅動架構重構。
