# 2026-03-06 開發日誌（Load DMA Parity Fix + AccuracyCoin 調查）

## 本次改動摘要

### 1. Load DMA countdown 改為 parity-dependent

**目標**：修復 3 個長期 FAIL 的 blargg 測試，達成 174/174 全通過。

**問題**：Load DMA（$4015 write 觸發）的 scheduling delay 固定為 3 個 APU cycle，
但真實硬體（及 Mesen2）使用 CPU cycle parity 來決定 delay 是 2 或 3。

**Mesen2 參考**（`DeltaModulationChannel.cpp` SetEnabled）：
```cpp
if((_console->GetCpu()->GetCycleCount() & 0x01) == 0) {
    _transferStartDelay = 2;
} else {
    _transferStartDelay = 3;
}
```

**修復**（`APU.cs`）：
```csharp
// 舊: dmcLoadDmaCountdown = 3;
// 新:
dmcLoadDmaCountdown = (apucycle & 1) != 0 ? 2 : 3;
```

Mapping: `apucycle` odd = Mesen2 CycleCount even → delay 2；
`apucycle` even = Mesen2 CycleCount odd → delay 3。

**修復的測試**：

| 測試 | 舊結果 | 新結果 |
|------|--------|--------|
| dmc_dma_during_read4/double_2007_read | FAIL | PASS |
| read_joy3/count_errors | FAIL | PASS |
| read_joy3/count_errors_fast | FAIL | PASS |

**結果**：171 PASS → **174 PASS / 0 FAIL / 174 TOTAL**

### 2. AccuracyCoin test 82 (IFlagLatency) 調查

**問題**：AccuracyCoin test 82 卡住不結束。

**根因分析**：
DMC DMA stolen cycle 計算使用 `cpuBusIsWrite` 作為 GET/PUT phase 的近似值。
在 AccuracyCoin 的 clockslide 序列中，79 個 DMA 有 9 個落在 JSR push（write cycle），
每個少算 1 個 stolen cycle，累計差 8 cycle，導致 DMA 無法對齊 $4013 opcode fetch。

**嘗試與結果**：

| 方案 | 結果 | 問題 |
|------|------|------|
| Deferred Reload DMA（全部延遲 1 cycle） | 172/174 | 破壞 sprdma 兩項 |
| Write-cycle only deferral | 173/174 | 破壞 sprdma_512 |
| Reload haltCycles=3 always | 172/174 | 破壞 sprdma |
| apucycle parity for Reload | 172/174 | 破壞 sprdma |
| 相反 parity | 168/174 | 破壞 6 項測試 |
| 統一 parity（無 Load/Reload 區分）| 168/174 | 破壞 6 項測試 |
| Global cpuCycleCount counter | 172/174 | 同 apucycle 結果 |

**結論**：修復需要真正的 GET/PUT phase tracking（用 `apucycle` parity 取代 `cpuBusIsWrite`），
但目前兩個 parity 方向都會破壞 sprdma 測試。根本原因是 `apucycle` 的初始 offset
與真實硬體的 phase 不匹配。這需要進一步校準初始 offset，屬於後續工作。

**架構評估**：Sub-cycle timing 改動本身不大（~15 行 APU.cs），但 parity offset
校準問題使得簡單套用會造成回歸。需要：
1. 引入 `cpuCycleCount` 獨立計數器（與 apucycle 區分）
2. 校準初始 phase 使 GET/PUT 對齊真實硬體
3. 驗證 sprdma + AccuracyCoin 同時通過
