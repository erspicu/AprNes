# 2026-02-26 Development Log (Performance Optimizations)

## Summary of Changes

### 1. Move SRAM I/O Out of NesCore

**Goal**: NesCore no longer reads or writes the file system directly; SRAM access is managed by the caller (UI layer).

**Approach**:
- `Main.cs`:
  - `battery` field changed to `public bool HasBattery`
  - Removed `rom_sav`, `SaveRam()`, and file I/O from `init()`
  - Added `LoadSRam(byte[] data)` (copies into `NES_MEM[0x6000..]`) and `DumpSRam()` (returns `byte[]`)
  - Removed `using System.IO`
- `AprNesUI.cs`: Added `SaveSRam()` / `LoadSRam()` helpers, called on ROM load, HardReset, and close

---

### 2. XInput Dual-API Controller Support

**Goal**: Support Xbox controllers via direct P/Invoke of the XInput API, without any third-party components.

**Approach**:
- `NativeAPIShare.cs`: Added `XINPUT_GAMEPAD`, `XINPUT_STATE` structs and `XInputGetState` DllImport (`xinput1_4.dll`)
- `joystick.cs`: Rewritten for WinMM + XInput dual-mode polling
  - XInput device ID = 1000 + player_index (avoids collision with WinMM range 0–255)
  - D-Pad → X/Y axis values 0/32767/65535, compatible with the existing key mapping format
  - Left analog stick supports dead zone (dead zone 8000)
  - Automatically falls back to WinMM-only when the DLL is not present

### 4. APU.cs 1D Managed Arrays → unsafe int\* / byte\* Pointers

**Goal**: Eliminate managed array bounds-check overhead on APU hot paths.

**Approach**:
- Added `using System.Runtime.InteropServices`
- `NesCore` class declaration changed to `unsafe partial class` (PPU.cs was already unsafe)
- 21 `int[]` fields → `int*`, allocated with `Marshal.AllocHGlobal`
- 9 `bool[]` fields → `byte*` (0 = false, 1 = true)
- `initSquareLookup()` / `initTndLookup()` helper functions removed; logic inlined into `initAPU()`
- `SQUARELOOKUP.Length` → constant `31`; `TNDLOOKUP.Length` → constant `203`
- Null-check pattern: `if (xxx == null) xxx = (int*)Marshal.AllocHGlobal(...)` prevents duplicate allocation

**Left unchanged**:
- `int[,] DUTYLOOKUP` (2D array)
- `int[] TRI_SEQ` (small read-only table; boundary access does not affect hot paths)

**Conversion list**:

| Original type | Field | Size |
|---------------|-------|------|
| `int[]` | `_pulseTimer/Period/Seq/Duty/Out` | 2 each |
| `int[]` | `volume` | 4 |
| `int[]` | `lengthctr`, `lengthctr_snapshot`, `lenctrload` | 4, 4, 32 |
| `int[]` | `envelopeValue/Counter/Pos` | 4 each |
| `int[]` | `sweepperiod/shift/pos` | 2 each |
| `int[]` | `noiseperiod`, `dmcperiods` | 16 each |
| `int[]` | `frameReload4/5` | 4, 5 |
| `int[]` | `SQUARELOOKUP`, `TNDLOOKUP` | 31, 203 |
| `bool[]` | `lenCtrEnable`, `lengthClockThisCycle`, `lenctrHalt` | 4 each |
| `bool[]` | `envConstVolume`, `envelopeStartFlag` | 4 each |

### 5. PPU NesColorsData / defaultPal Managed Arrays Removed

**Goal**: Remove the two remaining `static readonly` managed arrays from PPU.cs.

**Before**:
- `static readonly uint[] NesColorsData` (64 entries) → copied into `NesColors` (`uint*`) at startup
- `static readonly byte[] defaultPal` (32 entries) → copied into `ppu_ram[0x3F00..]` (`byte*`) at startup

**Approach**:
- Removed both managed array declarations
- Added `static void initPalette()` method that directly assigns values to `NesColors` (`uint*`) and `ppu_ram[0x3F00..]` (`byte*`) one by one
- `Main.cs init()` now calls `initPalette()`, replacing the two original for loops
