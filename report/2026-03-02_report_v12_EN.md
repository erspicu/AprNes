# 2026-03-02 Development Log (DirectInput Refactor + Controller Config Fixes)

## Summary of Changes

---

### 1. WinMM → DirectInput8 + XInput Refactor

**Goal**: The WinMM `joyGetPos` API is too outdated — it does not support XInput controllers, D-pad POV hats, and is no longer recommended.

**Approach**:
- Added `AprNes/tool/DirectInputHelper.cs`: implements DirectInput8 COM operations via raw vtable dispatch
- Rewrote `AprNes/tool/joystick.cs`: DirectInput (non-XInput) + XInput dual code paths
- Removed all WinMM P/Invoke declarations and structs from `NativeAPIShare.cs`

**Problem encountered**: `[ComImport]` RCW threw `AccessViolationException` when passing a managed delegate to `EnumDevices`

**Solution**: Switched to manual raw vtable dispatch + `Marshal.GetFunctionPointerForDelegate` to pass the callback as an IntPtr, completely bypassing the CLR COM marshaling mechanism.

---

### 2. Configuration UI Fix — Spurious "Non-Button Input" Warning

**Problem**: The configuration UI continuously popped up a "Non-Button input!" MessageBox even when no key was pressed.

**Cause**: DirectInput axes produce hardware noise events while at rest, and the configuration UI had no event-type filtering.

**Fix**:
- `AXIS_NOISE=256`: axis values within ±256 of center are normalized to 32767
- `ExpectedJoyInputType()`: A/B/Start/Select → type 1 (Button only); UP/DOWN/LEFT/RIGHT → type 2 (any)
- Configure polling silently filters events that do not match the expected type

---

### 3. Configuration UI Fix — Directional Fields Cannot Be Assigned (Defensive Improvement)

**Problem**: `Setup_JoyPad_define()` used `.Focused` to determine the currently active configuration field, which is unreliable under cross-thread Invoke + modal ShowDialog combinations.

**Fix**: Track the user's clicked field explicitly using an `_activeJoyControl` field + `Click`/`Enter` events, instead of relying on Windows focus state.

---

### 4. Directional Key Config Root Cause Fix — Imprecise Axis Values

**Problem**: JoyTest diagnostics confirmed that D-pad mode D produces `event_type=0 AXIS` events, but directional key configuration and in-game input still had no response.

**Root cause**: `JoyPadWayName()` required the axis value to be exactly `0` or `65535`. If `DIPROP_RANGE` setup failed or the controller returned a slightly off value (e.g. `1`, `65434`), the function returned an empty string, causing both configuration and keymap lookups to fail.

**Fix**: Added `SnapAxisDigital(v)`:
- `v < 16384` → `0`
- `v > 49151` → `65535`
- Otherwise → `32767`

Applied to all DirectInput / XInput axis values, ensuring `JoyPadWayName` always receives a canonical value.

---

### 5. JoyTest Diagnostic Tool

Added `tools/JoyTest.cs`, a standalone WinForms diagnostic tool for inspecting DirectInput / XInput device events (event_type, axis values, buttons, POV, raw values).
