### Technical Highlights

#### Per-cycle CPU Rewrite (122ŌåÆ126, +4)

Changed the CPU from an "execute all cycles for one instruction at once" model to a "step one cycle at a time" model.
Each CPU cycle is an independent `StartCpuCycle ŌåÆ bus op ŌåÆ EndCpuCycle`,
allowing DMA to be inserted at any cycle boundary for precise stolen-cycle simulation.

**Tests fixed**:
- P12 IFlagLatency: Test E no longer hangs (DMC DMA cycle drift eliminated)
- P20 Instruction Timing + Implied Dummy Reads: DMA pre-conditions now pass
- P13 DMA + $4015 Read / DMC DMA + OAM DMA: DMA timing aligned

#### SH* Unofficial Opcodes Fix (126ŌåÆ131, +5)

Correct DMA bus conflict behavior for SHA/SHX/SHY/SHS instructions.
When a DMA occurs before the SH* write cycle, value = A & X (H AND masking is removed).
All 6 P10 tests now PASS.

#### DMC DMA Cooldown (131ŌåÆ132, +1)

Based on TriCNES's `CannotRunDMCDMARightNow` mechanism,
a cooldown period is set after a DMC DMA completes to prevent back-to-back DMC DMA triggers.
