## Technical Highlights

### BUGFIX19: DMC DMA Cycle Stealing
The most challenging fix in this version. Originally marked as "requires architectural refactoring" in TODO.md, this was resolved without a major rewrite through the following design:

- **dmc_stolen_tick()**: A PPU-only tick function that advances only PPU 3 dots + NMI/IRQ tracking, without calling `apu_step()` to avoid recursion
- **Load/Reload model**: Based on the NESdev Wiki DMA reference, distinguishes between Load DMA triggered by `$4015` write (3 cycles) and Reload DMA triggered by buffer exhaustion (4 cycles), plus write-delay inversion rules
- **Phantom reads**: Simulates repeated CPU bus reads during DMC halt; $4016/$4017 only triggered on halt cycles (NES-001 /OE behavior)
- **--expected-crc**: New CRC comparison mechanism in TestRunner, supporting tests with multiple valid CRCs (due to CPU-PPU sync randomness)
