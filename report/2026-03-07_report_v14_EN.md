### Technical Highlights

#### BUGFIX45: $2002 Flag Clear Timing

The final fix resolved a problem previously believed to be "impossible to fix without regressions."

**Problem**: AccuracyCoin P18 test `$2002 flag clear timing` requires sprite flags (sprite 0 hit, sprite overflow) to clear earlier than the VBL flag.

**Previously failed attempts**:
- Dot 2→3 VBL stagger: regressed P17 VBlank End (-2)
- Unified clear at dot 1: no improvement

**Breakthrough**: Analysis of the AccuracyCoin source code revealed that P17 VBlank End only checks the VBL flag (bit 7), not the sprite flags. It is therefore safe to move sprite flag clearing forward to dot 1 while keeping VBL at dot 2.

```csharp
// Before: all flags at dot 2
if (scanline == 261 && ppu_cycles_x == 2)
    isVblank = isSprite0hit = isSpriteOverflow = false;

// After: sprite flags at dot 1, VBL at dot 2
if (scanline == 261 && ppu_cycles_x == 1)
    isSprite0hit = isSpriteOverflow = false;
if (scanline == 261 && ppu_cycles_x == 2)
    isVblank = false;
```

This simulates the real RP2A03G M2 duty cycle effect: sprite flags are read on M2 fall (later), while VBL is read on M2 rise (earlier), making sprite flags appear to clear ~2 PPU dots before VBL.
