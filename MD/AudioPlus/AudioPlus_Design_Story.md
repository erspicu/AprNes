# AudioPlus — AprNes 音訊引擎設計理念

## 我們想解決什麼問題？

NES 的音訊硬體誕生於 1983 年。五個聲道——兩組方波、一組三角波、一組雜音、一組 DPCM 取樣——以 1,789,773 Hz 的 CPU 時脈速率運作，比 CD 音質的 44,100 Hz 快了 40 倍。

大多數模擬器面臨的共同挑戰是：**如何把這個超高速的數位信號，正確地轉換成你耳朵聽到的聲音？**

最簡單的做法是每隔 ~41 個 CPU cycle 直接抓一個樣本。這就像用手機拍攝電視螢幕——你會看到摩爾紋。在音訊領域，這叫做**混疊失真（aliasing）**：原本不存在的假頻率出現在輸出中，方波聽起來刺耳、高音粗糙、整體帶有一層數位感的毛邊。

AprNes 的 AudioPlus 引擎，就是為了徹底解決這個問題而設計的。

---

## 三種模式，三種聆聽哲學

AudioPlus 提供三種音訊模式，對應三種不同的聆聽需求。不是「好壞」之分，而是「你想聽到什麼」的選擇。

### Mode 0：Pure Digital — 原始直出

**「給我最快、最乾淨、最沒有修飾的聲音。」**

這是最傳統的模擬器音訊路徑：五個聲道的輸出經過查找表混合，通過一個簡單的 DC 消除濾波器，直接送到你的喇叭。沒有超採樣，沒有額外濾波，沒有任何後處理。

為什麼有人會選擇這個模式？因為它的延遲最低、CPU 開銷最小，而且對很多人來說，這就是他們記憶中「NES 聽起來的樣子」——帶著一點粗糙的數位質感，但充滿力量。

### Mode 1：Authentic — 考究復古

**「我想聽到 1985 年那台 Famicom 從電視喇叭傳出來的聲音。」**

真實的 NES 不是把數位信號直接送到你耳朵的。它經過了 DAC（數位類比轉換器）、主機內部的電路濾波、輸出線材的衰減，最後才到達電視的喇叭。每一個環節都會改變聲音的特性。

Authentic 模式完整重現了這條路徑：

**物理級 DAC 模擬**

NES 的 DAC 不是簡單的線性加法。當 Pulse 1 輸出 7、Pulse 2 輸出 8 時，混合結果不是 7+8=15 的線性對應值——電阻梯級網路讓聲道之間產生非線性交互作用。我們使用 NESdev 社群記錄的精確 DAC 公式，預計算了一張 128KB 的 3D 查找表（Triangle × Noise × DMC 三個維度的完整組合），在每個 CPU cycle 查表取得精確的類比電壓值。

**256-Tap 超採樣降頻**

這是 Authentic 模式音質提升最大的關鍵。我們不像 Pure Digital 那樣每 41 cycle 粗暴地抓一個樣本，而是把 1.79 MHz 的完整波形全部記錄下來，再用一組 256 個係數的 FIR 濾波器（Blackman 窗函數加權的 sinc 函數），精確地從中萃取出 44.1 kHz 的音訊。這組濾波器的截止頻率設在 20 kHz——剛好是人耳聽覺的極限——確保所有可聽頻率完整保留，同時消除會造成混疊的超高頻成分。

為了處理 1,789,773 / 44,100 = 40.584... 這個非整數的降頻比，我們使用了 128 組多相位（polyphase）核心，每組對應不同的小數延遲，讓輸出樣本精確落在正確的時間點上，不產生相位誤差。

**主機型號濾波**

不同版本的 NES 硬體聽起來截然不同。初代 Famicom (HVC-001) 的 RF 輸出明亮清晰；美版 Front-Loader (NES-001) 的封閉式機殼讓高頻明顯衰減，聽起來溫暖厚實；後期的 Top-Loader (NES-101) 幾乎不濾波，聲音銳利但帶有 60 Hz 的交流電哼聲——那是 PCB 走線設計不良造成的。

我們內建了六款真實主機的濾波特性：

| 主機 | 截止頻率 | 聲音特徵 |
|------|---------|---------|
| Famicom (HVC-001) | 14 kHz | 明亮清晰，經典紅白機音色 |
| Front-Loader (NES-001) | 4.7 kHz | 溫暖厚實，很多西方玩家的童年記憶 |
| Top-Loader (NES-101) | 20 kHz | 銳利通透，伴隨輕微 AC 哼聲 |
| AV Famicom (HVC-101) | 19 kHz | 乾淨直出，後期改良版 |
| Sharp Twin Famicom | 12 kHz | 略暗於原版，Sharp 聯名機 |
| Sharp Famicom Titler | 16 kHz | S-Video 等級音質 |

還有第七個選項：完全自訂。你可以自由設定低通截止頻率（1,000–22,000 Hz）、交流電哼聲（50/60 Hz 及振幅）、RF 串擾程度，調出你記憶中那台主機的聲音。

**RF 串擾**

早期 Famicom 透過 RF 線輸出影音。RF 線材中的視訊信號會輕微串入音訊通道，產生一種隨畫面明暗變化的微弱雜音。大部分人不會刻意注意到它，但當你把它關掉，又會覺得「少了點什麼」。Authentic 模式可以選擇性地加入這個效果，使用 59.94 Hz 的鋸齒波乘以畫面亮度來模擬。

### Mode 2：Modern — 現代立體聲

**「NES 的音樂很棒，但我想用現代音響工程的手法重新詮釋它。」**

Modern 模式打破了硬體模擬的框架。它的核心思路是：**保留每個聲道的獨立性，讓它們各自發揮**。

**五軌獨立超採樣**

與 Authentic 模式「先混音、再降頻」不同，Modern 模式為五個聲道各配備一組獨立的 256-Tap FIR 降頻引擎。先降頻、再混音——這意味著每個聲道在進入混音台之前，都已經是乾淨的 44.1 kHz 信號，可以獨立控制。

為什麼要這麼做？因為一旦混在一起就無法分開了。就像錄音室裡，每個樂器有自己的麥克風和軌道，混音師才能自由調配平衡。

**立體聲分離**

原始 NES 是單聲道輸出。Modern 模式將五個聲道重新配置到立體聲音場中：

- Pulse 1 偏左、Pulse 2 偏右——兩組方波分居兩側，營造出寬度感
- Triangle、Noise、DMC 固定在中央——低頻和節奏錨定在中心，保持穩定

分離程度可調（StereoWidth 0–100%）。0% 就是純單聲道，100% 是最大分離。

**Triangle 低音增強**

NES 的 Triangle 聲道是一組 4-bit 步階三角波——只有 16 級電壓。這讓它在低頻段缺乏豐滿感，聽起來偏薄。我們用一組 Low-Shelf Biquad 均衡器（Audio EQ Cookbook 標準公式），專門針對 Triangle 聲道在 80–300 Hz 頻段施加 0–12 dB 的提升。只動 Triangle，不影響其他聲道——這就是獨立超採樣的好處。

**Haas Effect — 空間感**

人耳判斷聲源方位的關鍵之一是左右耳的到達時間差。當同一個聲音先到左耳、10–30 毫秒後到右耳時，大腦不會把它聽成回音，而是感覺聲源「更寬」、「更有空間感」。這就是 Haas Effect（優先效應）。

我們將右聲道延遲 10–30 ms（可調），並將延遲後的信號以一定比例混回左聲道（crossfeed），讓原本鎖在頭腦正中央的 NES 音樂擴展開來，特別適合用耳機聆聽。

**Micro-Room Reverb — 微型房間殘響**

四條平行的 Comb Filter（梳狀濾波器），延遲長度分別設定為 1116、1188、1277、1356 個樣本——這些數字互質，避免產生金屬腔體般的共振。每條 Comb Filter 的回饋路徑都帶有一階低通濾波，模擬真實房間中牆壁對高頻的吸收。

這不是音樂廳式的長殘響（那會讓 NES 快節奏的音樂變成一團糊），而是輕微的、像在小房間裡播放的空間感。濕度（wet）最高只到 30%，確保效果是加分而非干擾。

## 為什麼要加這些濾波器？每一個都有意義

AudioPlus 裡的每一個濾波器都不是隨意添加的。以下是它們各自存在的原因：

| 濾波器 | 位置 | 為什麼需要它 |
|--------|------|------------|
| 90 Hz 高通 (HPF) | Mode 0/1 | NES DAC 混音會產生直流偏移（DC offset），如果不消除，喇叭振膜會被推到一側，降低動態範圍並產生破音 |
| 256-Tap FIR 低通 | Mode 1/2 | Anti-aliasing——阻止 20 kHz 以上的超高頻在降頻時折返成可聽到的假頻率。這是數位信號處理的基本要求（Nyquist 定理） |
| 主機型號 IIR 低通 | Mode 1 | 真實硬體的類比電路本身就是一個低通濾波器，不同主機的截止頻率不同，決定了音色的明暗 |
| 60 Hz Buzz 正弦波 | Mode 1 | 模擬某些主機（如 Top-Loader）因 PCB 設計缺陷產生的交流電哼聲 |
| RF 鋸齒串擾 | Mode 1 | 模擬 RF 線材中視訊信號竄入音訊的現象，為聲音增添「那個年代」的質感 |
| Low-Shelf Biquad EQ | Mode 2 | 補償 Triangle 聲道 4-bit 步階波天生的低頻不足，讓貝斯線更飽滿 |
| Comb Filter + LPF 阻尼 | Mode 2 | 產生自然的房間殘響感。阻尼模擬高頻能量比低頻衰減更快的物理現象 |
| Haas 延遲線 | Mode 2 | 利用人耳的心理聲學特性，在不產生明顯回音的前提下擴展音場寬度 |

## 與 blargg's blip_buf 的設計差異

在 NES 模擬器音訊領域，blargg（Shay Green）的 **blip_buf** 是最廣為人知的音訊處理方案。它被 Mesen、Nestopia、Mednafen 等知名模擬器採用。了解 AprNes 的做法與 blip_buf 的差異，有助於理解我們為什麼走了一條不同的路。

### blip_buf 的做法：帶限步階合成（Band-Limited Step Synthesis）

blip_buf 的核心思想非常優雅：**只記錄變化，不記錄絕對值**。

當一個方波從 0 跳到 15 時，blip_buf 不是直接在緩衝區寫入一個瞬間跳變的垂直邊緣，而是注入一段預計算好的平滑 S 曲線（band-limited step），這段曲線是 sinc 函數的積分——在頻域上，它精確地只包含 Nyquist 頻率以下的成分，因此不會產生混疊。

在聲道安靜不變的時候，blip_buf 完全不做任何計算。只有在電壓發生變化的那一瞬間，它才注入一小段（約 16 個樣本長度的）脈衝。在一幀結束時，對所有累積的 delta 做一次積分，就得到了最終的波形。

這是一種**事件驅動**的架構：靜默時零開銷，只在波形跳變時才有計算量。

### AprNes 的做法：原生速率超採樣 + FIR 降頻

AprNes 走了一條不同的路：**記錄每一個 CPU cycle 的完整波形，再用重型 FIR 濾波器降頻**。

每個 APU cycle（1.789 MHz），我們都把當前的電壓值寫入一個 64K 容量的環形緩衝區。每累積約 40.58 個樣本後，用一組 256 個係數的 Blackman-windowed sinc FIR 與緩衝區做卷積，產出一個 44.1 kHz 的輸出樣本。

這是一種**暴力搜索**式的架構：不管聲道有沒有在發聲，每個 cycle 都寫入、每次降頻都做完整的 256-tap 卷積。

### 為什麼 AprNes 選擇超採樣而非 blip_buf？

| 面向 | blip_buf | AprNes 超採樣 |
|------|----------|--------------|
| 抗混疊品質 | 16-tap 脈衝（旁瓣衰減約 40 dB） | 256-tap Blackman sinc（旁瓣衰減 > 58 dB） |
| 靜默時效率 | 極高（零計算） | 固定開銷（但 SIMD 攤平後極低） |
| 每聲道獨立處理 | 需要為每聲道各建一個 blip buffer | 天然支援（每聲道一個 ring buffer） |
| 後處理彈性 | 低（delta 域，難以施加頻域濾波） | 高（標準 PCM 流，可自由接 EQ、reverb、pan） |
| 非方波聲道 | 需特殊處理（Triangle 的緩慢斜坡不適合 delta 模型） | 無差別處理（所有聲道統一） |
| SIMD 友好度 | 低（散列式 delta 注入，難以向量化） | 高（256-tap 連續卷積，Vector\<float\> 直接加速） |
| 擴展音效整合 | 需為每種 Mapper 音源改寫 blip 接口 | 直接加入原生速率緩衝區，統一降頻 |

AprNes 選擇超採樣的核心理由有三個：

**1. 更高的濾波品質**

blip_buf 使用 16 個樣本長度的脈衝（half_width=8），旁瓣衰減約 40 dB。AprNes 的 256-tap Blackman FIR 旁瓣衰減超過 58 dB——這意味著殘餘的混疊能量低了近 100 倍。在聽感上，高頻段更乾淨、方波的邊緣更平滑。

**2. 後處理的彈性**

AprNes 的三模式架構需要對音訊做大量後處理：主機型號濾波、立體聲 pan、Triangle EQ、Haas delay、Comb reverb。這些操作都需要在標準 PCM 時域信號上進行。blip_buf 的 delta 累積模型在積分之前是變化量而非絕對值，不太適合直接施加這些處理。超採樣的輸出是標準的浮點 PCM 流，可以自由串接任何 DSP 模組。

**3. 現代硬體上的 SIMD 優勢**

在 2020 年代的 CPU 上，256-tap FIR 卷積可以用 AVX2 指令一次處理 8 個浮點數，256 個 tap 只需要 32 次 SIMD 運算。一幀（735 個輸出樣本）的降頻計算量在毫秒等級以內。blip_buf 的 delta 注入是散列式的記憶體訪問模式，反而較難利用 SIMD 和快取局部性。

在 NES 這個量級的模擬上，「靜默時零開銷」的優勢已經被現代 CPU 的絕對算力稀釋——即使每個 cycle 都做寫入，也就是每秒 179 萬次浮點寫入加每秒 44100 次 256-tap 卷積，遠在現代處理器的能力範圍內。

### 總結

blip_buf 是一個為 2000 年代硬體條件設計的精巧演算法，在資源受限的環境下提供了極佳的效率/品質平衡。AprNes 的超採樣引擎則是為現代硬體量身打造——用更高的絕對計算量換取更好的濾波品質、更大的架構彈性、更友善的 SIMD 並行性。兩者殊途同歸，都是為了解決同一個根本問題：讓 1.789 MHz 的數位方波變成你耳朵聽到的乾淨音樂。
