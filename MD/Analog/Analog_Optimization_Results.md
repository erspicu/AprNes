## 結論

**優化週期完成**。16 個候選項目中：
- **2 項採用**: B8 (+1.51%), B6 (+4.86%) — 皆為 pass 合併，減少記憶體頻寬壓力
- **8 項實測退回**: 微優化（預計算、展開、定點化、分支消除）在 .NET JIT 下效益極小或反而更慢
- **6 項不適用**: Level 2 路徑（A2/A3/A6）、非 N=4 模式（B1/B7）、無優化空間（C2）

**未測項目**: A1（高複雜度，需重構主迴圈）、B2/B3/B4（需 System.Runtime.Intrinsics，.NET Framework 4.8.1 支援有限）、C3（在 B6 合併 pass 中，額外邏輯增加複雜度）。這些項目的收益/風險比不足以繼續測試。

**關鍵發現**: 在此管線中，**記憶體頻寬是主要瓶頸**——pass 合併（減少全幀遍歷次數）是唯一有效的優化手段。計算層面的微優化（除法消除、FIR 展開、定點化等）對效能無明顯影響，因為 .NET JIT 已做了足夠好的最佳化。
