實作 **Sunsoft 5B** 的音效核心（YM2149/PSG）時，最精確的方法是使用一個 **對數振幅查找表 (Logarithmic Lookup Table)**。

這是因為 YM2149 的 16 階音量並非線性增加，而是為了模擬人類耳朵對聲音強度的感知，採用了對數曲線。

### **1\. YM2149 16 階對數振幅表**

在您的 C\# 混音器中，請建立以下靜態數組。這些數值代表了 YM2149 內部 DAC 的輸出電平：

C\#

// YM2149 16-step Logarithmic Volume Table  
// 數值範圍 0.0 到 1.0 (float)  
private static readonly float\[\] YM2149\_VolumeTable \= new float\[16\] {  
    0.0000f, 0.0100f, 0.0150f, 0.0225f,   
    0.0338f, 0.0506f, 0.0759f, 0.1139f,   
    0.1709f, 0.2563f, 0.3845f, 0.5767f,   
    0.8651f, 1.2976f, 1.9464f, 2.9197f // 最終輸出會再除以最大值進行歸一化  
};

**技術註解**：YM2149 的每一階音量大約比前一階增加 **1.5 倍 (約 3.5dB)**。

### ---

**2\. 正確的混音與輸出邏輯**

當您從 Mapper 069 讀取到 3 個頻道的狀態時，請依照下列公式計算輸出：

C\#

public float Get5BOutput(int chanA\_vol, int chanB\_vol, int chanC\_vol, bool maskA, bool maskB, bool maskC) {  
    float outA \= maskA ? 0 : YM2149\_VolumeTable\[chanA\_vol & 0x0F\];  
    float outB \= maskB ? 0 : YM2149\_VolumeTable\[chanB\_vol & 0x0F\];  
    float outC \= maskC ? 0 : YM2149\_VolumeTable\[chanC\_vol & 0x0F\];

    // 將 3 個頻道加總，並乘上相對於 NES APU 的增益係數  
    // 建議 Gain 設定在 0.2 \~ 0.25 之間，以符合實機聽感  
    return (outA \+ outB \+ outC) \* 0.22f;   
}

### ---

**3\. 實作中的細節陷阱**

1. **高通濾波器 (DC Bias)**：YM2149 的輸出通常帶有直流偏移，如果直接加到 APU 輸出，可能會導致音訊剪裁（Clipping）。建議在最後輸出前套用一個簡單的濾波器。  
2. **時脈同步**：5B 的內部計數器是根據 CPU Cycle 驅動的。在 **Dendy 模式** 下，由於 CPU 頻率（1.77MHz）與 NTSC（1.79MHz）非常接近，音調偏差極小，幾乎聽不出來。  
3. **封絡控制 (Envelope)**：YM2149 還有一個硬體封絡產生器（Register 11-13）。雖然《Gimmick\!》主要使用手動音量控制，但若要完美支援 138 個 Mapper，這部分的實作也不可缺少。
