# BUGFIX33: AccuracyCoin Phase 1 - Controller strobe, OAMADDR reset, S0H rendering flags

## Date: 2026-03-06

## Baseline: 171 PASS (legacy tests), 106/136 AccuracyCoin PASS

## Fix Details

### 1. Controller Strobing bit 0 check (JoyPad.cs)
- **Problem**: $4016 strobe only checked `P1_LastWrite == 1`, but NES hardware checks bit 0
- **Fix**: Changed to `(P1_LastWrite & 1) == 1 && (val & 1) == 0`
- **Effect**: AccuracyCoin Controller Strobing improved from err=2 to err=4 (3/4 subtests pass)
- Test 4 (put/get cycle parity) still FAIL, requires CPU bus cycle parity tracking

### 2. OAMADDR reset during rendering (PPU.cs)
- **Problem**: spr_ram_add (OAMADDR) was not reset during rendering, causing address offset after OAM DMA
- **Root Cause**: NES hardware resets OAMADDR to 0 at dot 257 of every scanline
- **Fix**: Added `spr_ram_add = 0` at dot 257 in ppu_rendering_tick()
- **Impact**:
  - Rendering Flag Behavior: FAIL → PASS (BG shift registers work correctly in sprites-only mode)
  - $2004 behavior: err=1 → err=4 (basic OAM write + DMA tests pass)
  - Fixed prerequisites for multiple sprite zero related tests

### 3. Sprite 0 hit detection flags (PPU.cs)
- **Problem**: PrecomputeSprite0Line required both flags enabled before precomputing (||)
- **Fix**: Changed to precompute when either flag is enabled (&&), but per-pixel detection requires both flags
  - PrecomputeSprite0Line: `!ShowBG || !ShowSpr` → `!ShowBG && !ShowSpr`
  - Per-pixel S0H: `renderingEnabled` → `ShowBackGround && ShowSprites`
- **Reason**: Data can be precomputed in sprites-only mode, but hit only triggers when both are enabled

## Test Results

- Legacy tests: 174/174 PASS (no regression)
- AccuracyCoin: 107/136 PASS (+1), 28 FAIL, 1 SKIP
