# 2026-03-19 Mapper034 / Mapper078 新增 + CRC 修正

## 變更摘要

1. 新增 Mapper034 (Nina-1)
2. 新增 Mapper078 (Irem 74HC161/32)
3. 修正 CRC32 計算方式（改為 PRG+CHR 區段，符合 Mesen2 格式）
4. 更新 MMC3 子變體 CRC 對應表
5. 新增 CHR ROM bank 數量上限保護
6. 修正 Mapper010 PRG bank wrapping

---

## 1. Mapper034 (Nina-1)

**檔案**: `AprNes/NesCore/Mapper/Mapper034.cs`

兩種子變體共用一個 Mapper 編號：

| 子變體 | 遊戲 | 暫存器位置 | PRG/CHR |
|--------|------|-----------|---------|
| CHR-RAM | Deadly Towers, Mashou | $8000-$FFFF (32K PRG) | CHR-RAM |
| CHR-ROM | Impossible Mission II | $7FFD/$7FFE/$7FFF | CHR-ROM (4K×2) |

- `MapperW_PRG($8000+)`: 僅 CHR-RAM 變體使用，選擇 32K PRG bank
- `MapperW_RAM($7FFD)`: 32K PRG 選擇（CHR-ROM 變體）
- `MapperW_RAM($7FFE)`: 低 4K CHR bank 選擇
- `MapperW_RAM($7FFF)`: 高 4K CHR bank 選擇
- 無 IRQ，無鏡像控制（固定依 ROM header）

## 2. Mapper078 (Irem 74HC161/32)

**檔案**: `AprNes/NesCore/Mapper/Mapper078.cs`

兩種子變體，透過 CRC 辨識：

| 子變體 | 遊戲 | CRC (PRG+CHR) | 鏡像 |
|--------|------|---------------|------|
| submapper 1 (預設) | Uchuusen Cosmo Carrier | — | 單畫面 A/B 切換 |
| submapper 3 (Holy Diver) | Holy Diver (J) | 0xBA51AC6F | V/H 切換 |

暫存器 `$8000-$FFFF`，格式 `CCCCPPPP`：
- bits 0-2: 16K PRG bank at $8000
- bit 3: 鏡像（Holy Diver: 0=H/1=V；Uchuusen: 0=single-A/1=single-B）
- bits 4-7: 8K CHR bank

**注意**: Uchuusen ROM header 誤標 four-screen flag，`Reset()` 時強制設為 single-A 覆蓋。

## 3. CRC32 計算方式修正

**檔案**: `AprNes/NesCore/Main.cs`

原本計算整個 ROM 檔案（含 16-byte header）的 CRC32，改為只計算 PRG+CHR 資料區段（跳過 16-byte iNES header，並考慮 trainer offset），符合 Mesen2 資料庫格式。

```csharp
// 舊：從 rom_bytes[0] 開始（含 header）
// 新：從 prgChrStart = 16 + trainerOffset 開始
```

## 5. CHR ROM bank 數量上限保護

**檔案**: `AprNes/NesCore/Main.cs`

部分 ROM（如 Famicom Wars (J)）header 宣告的 CHR bank 數量多於檔案實際內容，會導致越界讀取。新增 clamp 邏輯：

```csharp
int chrOffset = PRG_ROM_count * 16384 + 16;
int maxChrBanks = (rom_bytes.Length - chrOffset) / 8192;
if (CHR_ROM_count > maxChrBanks)
    CHR_ROM_count = (byte)maxChrBanks;
```

## 6. Mapper010 PRG bank wrapping

**檔案**: `AprNes/NesCore/Mapper/Mapper010.cs`

`MapperR_RPG` 存取時加上 `prgBank % PRG_ROM_count`，防止 bank index 超出 ROM 大小導致越界。
