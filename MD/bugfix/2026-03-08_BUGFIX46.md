# BUGFIX46: AccuracyCoin P14 APU Register Activation

**日期**: 2026-03-08
**基線**: 118/136 → **119/136** (+1)
**修復測試**: P14 Test 7 (APU Register Activation) err=6 → PASS

## 根因

P14 Test 7 驗證 OAM DMA 從 page $40 (APU register space) 讀取時的正確行為。
當 CPU 地址匯流排在 $4000-$401F 時，DMA 可讀取 APU 內部暫存器，
且每 $20 bytes 重複映射（$4036 → $4016 等）。

兩個問題：

### 1. IO_read 缺少 $4017 讀取處理

`IO_read()` 沒有 $4017 (controller 2) 的 case，導致 $4017 讀取
返回 `cpubus`（write-only 預設值）而非 controller 2 資料。
未接 controller 2 時應返回 D0-D4=0, D5-D7=open bus。

### 2. ProcessDmaRead bus conflict 合併前未更新 open bus

DMA 讀取 $4036 時映射到內部 $4016（controller 1），然後與外部匯流排
做 bus conflict AND 合併。外部匯流排 ($4020-$5FFF) 返回 open bus (cpubus)。

問題：cpubus 在內部讀取後未更新，仍保留前一次值 ($04 from $4015)。
導致 controller 值 $01 與 $04 做 AND = $00（應為 $01）。

Mesen2 的 `Read()` 每次都更新 open bus，所以外部讀取會看到更新後的值。

## 修復

### IO.cs
```csharp
case 0x4017: return (byte)(cpubus & 0xE0); // Controller 2: D0-D4=0, D5-D7=open bus
```

### MEM.cs (ProcessDmaRead)
在 $4015 和 $4016/$4017 case 中，external bus read 前先更新 cpubus：
```csharp
cpubus = val; // Update open bus before external read
```
