## 一、Sync Interop 優化（FPS 70 → ~100）

### 問題
- JS 使用 `invokeMethodAsync`（回傳 Promise），每幀產生 microtask 排程 overhead
- `JS.InvokeVoidAsync("drawFrame", rgba)` 將 245KB byte[] 做 Base64 JSON 序列化，再傳給 JS 解碼
- `OnFrame()` 為 `async Task`，每幀有 async 狀態機分配成本

### 修復

**`nesInterop.js`**
- `invokeMethodAsync` → `invokeMethod`（同步直呼叫，無 Promise）

**`NesJsInterop.cs`**（新增）
```csharp
[JSImport("globalThis.nesInterop.drawFrameUnmarshalled")]
internal static partial void DrawFrame(byte[] pixels);
```
使用 `[JSImport]` 同步呼叫取代 `JS.InvokeVoidAsync`

**`NesFrameStep.cs`**
```csharp
static readonly byte[] _rgbaBuffer = new byte[256 * 240 * 4];
```
重複使用靜態 buffer，消除每幀 GC 壓力

**`Home.razor`**
- `OnFrame()` 改為 `void`（同步），消除 async 狀態機 overhead

### 效果
| 狀態 | FPS（無限制模式） |
|------|----------------|
| 修復前 | ~70 |
| 修復後 | ~100 |

## 二、遊戲重啟 Loop 雙跑問題

### 問題
停止遊戲後再載入新遊戲，舊的 JS loop 沒有真正停止。  
`_romLoaded` 被新遊戲設為 `true` 後，舊 loop 重新開始執行幀，導致：
- 畫面 FPS 翻倍（解除限制）
- 音效與畫面嚴重不同步

### 根本原因
`startLoop` 回傳 `{ stop() }` 物件，但 Blazor 端沒有儲存並呼叫 `stop()`。

### 修復
在 `nesInterop.js` 用 **generation counter** 自動失效舊 loop：

```javascript
let loopGeneration = 0;

function startLoop(dotNetRef) {
    loopGeneration++;              // 令所有舊 loop 失效
    const myGen = loopGeneration;
    nextTime = 0;                  // 重置音效時間軸
    function loop() {
        if (loopGeneration !== myGen) return; // 舊 loop，停止
        dotNetRef.invokeMethod('OnFrame');
        ...
    }
}

function stopLoop() { loopGeneration++; }
```

`Home.razor` `StopGame()` 改為 async，先呼叫 `nesInterop.stopLoop()`：

```csharp
async Task StopGame() {
    NesCore.exit = true;
    await JS.InvokeVoidAsync("nesInterop.stopLoop");
    _romLoaded = false;
    ...
}
```

## 三、HTML5 Gamepad API 手把支援

### 功能
- 支援 XInput（Xbox 系列）及 DirectInput 手把
- 標準手把按鍵映射：

| NES | Gamepad |
|-----|---------|
| A | buttons[0]（A/○） |
| B | buttons[1] 或 buttons[2]（B/X） |
| Select | buttons[8] |
| Start | buttons[9] |
| 方向鍵 | buttons[12-15] 或左搖桿 axes[0/1] |

- 每幀在 `OnFrame()` 以差分方式更新（只在狀態改變時呼叫 Press/UnPress）
- 監聽 `gamepadconnected` / `gamepaddisconnected` 事件，即時顯示連接狀態

### 注意
瀏覽器安全限制：手把插入後需先**按任意鍵**，`gamepadconnected` 事件才會觸發。
