# BUGFIX 25 – Joystick Configuration UI Series of Fixes

**Date**: 2026-03-02
**Version**: AprNes
**Category**: Bug Fix

---

## Issue 1: Configuration UI Pops Up "Non-Button Input!" Warning

### Symptom

After opening the joystick configuration UI, `MessageBox` dialogs showing "Non-Button Input!" continuously appear even without pressing any button.

### Root Cause

DirectInput axis values have hardware noise when at rest (not precisely 32767), causing continuous axis events to be generated.
The A/B/Start/Select fields in the configuration UI only expect Button events but have no filtering mechanism, so axis events flow in and trigger the warning.

### Fix

1. **`joystick.cs`**: Added `AXIS_NOISE=256` — values within ±256 of center are forced to 32767, eliminating idle noise.
2. **`AprNes_ConfigureUI.cs`**: Added `ExpectedJoyInputType()` method:
   - A/B/Start/Select → returns `1` (accept Button only)
   - UP/DOWN/LEFT/RIGHT → returns `2` (accept any type)
   - No focus → returns `-1`
3. **`AprNesUI.cs`**: In configuration mode, filter events based on `ExpectedJoyInputType()`; silently ignore events with wrong type.

## Issue 2: D-pad Directions Cannot Be Configured in D Mode

### Symptom

After clicking the "Up" field in the configuration UI and pressing D-pad Up on the gamepad, the field shows no reaction.

### Root Cause (direct cause)

`Setup_JoyPad_define()` and `ExpectedJoyInputType()` use `control.Focused` to determine the currently focused field.
In a cross-thread `Invoke` + modal `ShowDialog` combination, the `Focused` property may not correctly reflect the field the user clicked when the Invoke lambda executes (cross-thread timing race condition).

### Fix (defensive improvement)

Added `_activeJoyControl` field, explicitly tracking the last field clicked by the user via `Click`/`Enter` events, without relying on `Focused`:

```csharp
TextBox _activeJoyControl = null;

void RegisterJoyActivation()
{
    foreach (TextBox tb in new[] { joypad_A, joypad_B, joypad_START, joypad_SELECT,
                                   joypad_UP, joypad_DOWN, joypad_LEFT, joypad_RIGHT })
    {
        TextBox captured = tb;
        captured.Click += (s, e) => _activeJoyControl = captured;
        captured.Enter += (s, e) => _activeJoyControl = captured;
    }
}
```

`ExpectedJoyInputType()` and `Setup_JoyPad_define()` now both use `_activeJoyControl` for comparison.

## Issue 3: D-pad Direction Keys Configured but Not Working In-Game (Button Mode Support)

### Symptom

Some gamepads report D-pad in D mode as button events (not axis), causing configuration to succeed but no response in game (or vice versa).

### Fix

- `ExpectedJoyInputType()` returns `2` for direction fields (accepts axis OR button)
- Game path (AprNesUI.cs gaming switch) added: direction button `button_event==0` fires UnPress
