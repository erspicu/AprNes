# BUGFIX43: Rendering Flag Behavior — Freeze BG Shift Registers When Rendering Off

**AccuracyCoin**: 116/136 -> 117/136 (+1 PASS)
**blargg**: 174/174 (no regression)

## Fix: BG Shift Register Freeze + Conditional Serial Input (P16, $0486, PASS)

### Problem
The "Rendering Flag Behavior" test (P16) checks three scenarios:
1. Disable all rendering → re-enable sprites only → sprite 0 hit should NOT occur
2. Disable BG only (sprites still on) → hit SHOULD occur (serial input fills high plane)
3. Disable sprites early, re-enable before hit → hit SHOULD occur

Test 1 failed because our BG shift registers unconditionally shifted with serial input
(high plane = 1) on every visible dot, even when all rendering was off. This filled the
high plane with 1s, creating a false non-transparent BG pixel → false sprite 0 hit.

### Root Cause
Two incorrect behaviors in the BG shift register shadow registers (lowshift_s0, highshift_s0):

1. **Unconditional shifting**: Shift registers shifted on every dot 0-255 regardless of
   rendering state. AccuracyCoin confirms: "shift registers are not clocked when rendering
   is disabled" (Stale BG Shift Registers test comment).

2. **Unconditional serial input**: High plane always received `| 1` serial input, even
   when all rendering was off. This caused empty shift registers to fill with non-zero
   data, producing false sprite 0 hits.

### Fix (PPU.cs: ppu_step_new)

Changed shift register behavior to be conditional on `renderingEnabled`:

**Before:**
```csharp
if ((scanline >= 0 && scanline < 240 && ppu_cycles_x < 256)
    || ((scanline < 240 || scanline == 261) && renderingEnabled
        && ppu_cycles_x >= 320 && ppu_cycles_x < 336))
{
    lowshift_s0 <<= 1;
    highshift_s0 = (ushort)((highshift_s0 << 1) | 1);  // always serial input
}
```

**After:**
```csharp
if (renderingEnabled)
{
    if ((scanline >= 0 && scanline < 240 && ppu_cycles_x < 256)
        || ((scanline < 240 || scanline == 261)
            && ppu_cycles_x >= 320 && ppu_cycles_x < 336))
    {
        lowshift_s0 <<= 1;
        highshift_s0 = (ushort)((highshift_s0 << 1) | 1);
    }
}
```

This ensures:
- **Rendering off**: Shift registers frozen → stale data preserved (P19 Stale BG test OK)
- **Rendering on**: Serial input applied → high plane fills with 1s (P16 Test 2 OK)
- **Never loaded**: Shift registers stay at initial 0 → no false hit (P16 Test 1 OK)

### Files Modified
- `PPU.cs`: Made BG shift register shifting conditional on `renderingEnabled`
