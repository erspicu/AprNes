# BUGFIX15 — read_joy3/thorough_test Script Fix

**Date**: 2026-02-22
**Affected Tests**: read_joy3/thorough_test (+1 PASS)
**Baseline Change**: 158 → 159 PASS / 16 → 15 FAIL

## Problem Analysis

`read_joy3/thorough_test.nes` had two parameter errors in the test script:

### Issue 1: Incorrect `--input A:2.0`
- The test uses DMC DMA interference to test joypad reads, verifying the `read_joy` multi-read fault-tolerance mechanism
- The test expects all read results to be 0 (no button pressed), checked at the end with `jne test_failed`
- `--input A:2.0` presses the A button at frame ~121, causing some iterations to read A=pressed → "Failed"
- This test requires no button input at all

### Issue 2: Insufficient timeout `--max-wait 30`
- The test runs 1000 iterations, each containing `sync_dmc` + variable delay + `read_joy`
- On our emulator it takes ~36.4 seconds to complete (frame 2189)
- `--max-wait 30` causes a timeout

## Fix

**run_tests_report.sh** and **run_tests.sh**:
```bash
# Before
run_test "read_joy3" "thorough_test.nes" "--max-wait 30 --input A:2.0"

# After
run_test "read_joy3" "thorough_test.nes" "--max-wait 45"
```

## Verification

- `read_joy3/thorough_test.nes`: PASS (frame 2189, ~36.4 seconds)
- Full test suite: 159 PASS / 15 FAIL / 174 TOTAL
- No regressions

## Technical Notes

- thorough_test purpose: verify that DMC DMA interference on $4016 reads does not exceed 1 in 4 reads
- Our emulator has no DMC DMA cycle stealing (Bug F), so reads are never interrupted
- Test still passes (0 interruptions < threshold of 2)
