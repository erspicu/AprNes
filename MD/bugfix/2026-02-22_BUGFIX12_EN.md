## 37. PPU VBL/NMI Timing Fix — 1-Cycle NMI Delay Model

### Symptom

| Test ROM | Error |

| ppu_vbl_nmi/10-even_odd_timing | "Clock is skipped too late" (#3) |
| vbl_nmi_timing/5-nmi_suppression | NMI suppression window |
| vbl_nmi_timing/6-nmi_disable | NMI disable timing |
| vbl_nmi_timing/7-nmi_timing | NMI trigger timing |

### Root Cause

#### 1. Missing NMI Edge Detection Model

The original implementation directly set `nmi_pending` inside `ppu_step_new()`, lacking a hardware-level edge detection mechanism. On real NES, NMI is edge-sensitive: the PPU outputs `nmi_output = isVblank && NMIable`; the CPU detects the rising edge and delays 1 CPU cycle before triggering NMI.

#### 2. VBL Clear Time Off

VBL flag cleared at `(sl=261, cx=1)` but the correct value is `(sl=261, cx=2)`, causing VBL to clear 1 PPU dot earlier than hardware.

#### 3. $2002 Read Missing VBL Flag Suppression

Reading $2002 at the same PPU dot `(sl=241, cx=1)` where VBL is set should return VBL=0 (flag suppressed), but the flag has already been set so it was being returned as set.

#### 4. NMI Cancellation Semantics Wrong

- Both `$2002` reads and `$2000` writes directly cleared `nmi_pending`
- But hardware behavior is: once NMI has completed its 1-cycle delay and been promoted to pending state, **$2002 reads cannot cancel it**
- `$2000` NMI disable can cancel a promoted NMI, but only if it hasn't yet been serviced at an instruction boundary

### Fix — 1-Cycle NMI Delay Model

Core concept: introduce `nmi_delay` intermediate state, modeling the hardware's 1 CPU cycle delay:

```
Rising edge detected → nmi_delay = true → next tick() promotes → nmi_pending = true → NMI fires
```

#### 1. MEM.cs — Rewrite tick() for Per-Dot Edge Detection + Delayed Promotion

```csharp
static void tick()
{
    if (in_tick) return;
    in_tick = true;

    // Promote nmi_delay from previous cycle → nmi_pending (1-cycle hardware delay)
    if (nmi_delay) { nmi_pending = true; nmi_delay = false; }

    // Per-dot NMI edge detection: rising edge → nmi_delay (not nmi_pending)
    for (int i = 0; i < 3; i++)
    {
        ppu_step_new();
        bool nmi_output = isVblank && NMIable;
        if (nmi_output && !nmi_output_prev)
            nmi_delay = true;       // Rising edge → 1-cycle delay before pending
        nmi_output_prev = nmi_output;
    }

    apu_step();
    in_tick = false;
}
```

**Key design**:
- Per-dot edge detection: check `nmi_output` after each PPU dot (not once at the end of tick)
- Rising edge sets `nmi_delay` (not `nmi_pending` directly)
- Each tick promotes `nmi_delay` → `nmi_pending` at the start (1 CPU cycle delay)

#### 2. PPU.cs — Fix VBL Clear

```csharp
// Old:
if (scanline == 261 && ppu_cycles_x == 1)
    isVblank = isSprite0hit = isSpriteOverflow = false;

// New:
if (scanline == 261 && ppu_cycles_x == 2)
    isVblank = isSprite0hit = isSpriteOverflow = false;
```

#### 3. PPU.cs — ppu_r_2002() VBL Suppression + NMI Cancel Semantics

```csharp
static byte ppu_r_2002()
{
    bool vblFlag = isVblank;

    // VBL suppression: reading at same dot as VBL set → return flag as 0
    if (scanline == 241 && ppu_cycles_x == 1)
        vblFlag = false;

    openbus = (byte)((vblFlag ? 0x80 : 0) | ...);

    isVblank = false;
    nmi_delay = false;         // cancel unpromoted NMI (same-cycle edge)
    nmi_output_prev = false;   // reset edge state
    // Note: do NOT clear nmi_pending — once promoted, $2002 read cannot cancel
    vram_latch = false;
    return openbus;
}
```

#### 4. PPU.cs — ppu_w_2000() Falling Edge Semantics

```csharp
// Falling edge (disable NMI):
// - clear nmi_delay (unpromoted can be cancelled)
// - do NOT clear nmi_pending (promoted cannot be cancelled)
bool nmi_output = isVblank && NMIable;
if (!nmi_output && nmi_output_prev)
{
    nmi_delay = false;
    nmi_output_prev = false;
}
```

#### 5. PPU.cs — Simplify ppu_step_new() VBL Set

```csharp
if (scanline == 241 && ppu_cycles_x == 1)
{
    if (!SuppressVbl)
        isVblank = true;
    // No longer directly set nmi_pending — handled by tick() edge detection
    SuppressVbl = false;
}
```

### Timing Verification

#### test 06 (suppression) — Key Difference Between Offset 05 vs 07

**offset 05 (V -, correctly no trigger)**:
1. tick() dot 1: VBL set → nmi_delay = true
2. Same cycle $2002 read: clears nmi_delay → no promotion → no NMI ✓

**offset 07 (V N, correctly triggers)**:
1. Cycle N tick(): VBL set → nmi_delay = true
2. Cycle N+1 tick(): promote nmi_delay → nmi_pending = true
3. $2002 read: clears nmi_delay (already false), **does not clear** nmi_pending → NMI fires ✓

#### test 08 (nmi_off_timing) — Key Difference Between Offset 06 vs 07

**offset 06 (-, correctly no trigger)**:
1. Same cycle: VBL set → nmi_delay = true
2. $2000 disable: clears nmi_delay → no promotion → no NMI ✓

**offset 07 (N, correctly triggers)**:
1. Cycle N: VBL set → nmi_delay = true
2. Cycle N+1: promote → nmi_pending = true
3. $2000 disable: only clears nmi_delay (already false), **does not clear** nmi_pending → NMI fires ✓
