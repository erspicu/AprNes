## 15. Sprite Priority Quirk: Mask Sprites Cannot Correctly Hide Foreground Sprites with High OAM Indices

**Commit**: `3b86e23`

### Symptom

Two sprite-penetrating-tile issues in SMB3:

- **Piranha Plant** body visible through the pipe mid-section while rising/descending
- **Mushroom power-up** visible through the block while emerging from a "?" block or brick

These are scenes where SMB3 uses the "mask sprite trick."
On real NES hardware, a lower-OAM-index behind-background sprite (priority=1) should occlude a higher-OAM-index foreground sprite at the same pixel position.

### Correct NES Hardware Behavior (Sprite Priority Quirk)

NES PPU sprite compositing is done in **two independent stages**:

**Stage 1: Sprite competition (OAM index priority)**

Each screen pixel is "won" by the sprite with the lowest OAM index that has an opaque pixel at that position.

**Stage 2: Compositing with background (determined by the winner's priority bit)**

The winning sprite's OAM byte 2, bit 5 determines the final output:

| Background pixel | Winning sprite pixel | Priority bit | Output |
|------------------|----------------------|--------------|--------|
| Transparent (0) | Opaque | Any | Sprite |
| Opaque | Opaque | 0 (foreground) | Sprite |
| Opaque | Opaque | 1 (behind) | Background |

**Critical quirk**: when a lower-OAM-index behind-background sprite (priority=1) wins, its priority=1 causes an opaque background to occlude **the entire pixel** — including the higher-OAM-index foreground sprite (priority=0) that would otherwise be visible.

### SMB3's Mask Sprite Trick

```
OAM index 2 = mask sprite (priority=1, opaque, placed at pipe/block position)
OAM index 5 = Piranha Plant sprite (priority=0, visible)
```

OAM 2 mask "wins" → its priority=1 + opaque background → entire pixel shows background → Piranha Plant is also hidden.

### Root Cause

The old code let each sprite **independently** use its own priority bit to decide whether to draw:

```csharp
// Before: reverse OAM loop, each sprite draws directly to screen
for (int si = selCount - 1; si >= 0; si--)
{
    // ...
    // each sprite judges its own priority independently
    if (pixel != 0 && (!ShowBackGround || Buffer_BG_array[array_loc] == 0 || !priority))
        ScreenBuf1x[array_loc] = NesColors[...];
}
```

Execution flow (wrong):

1. `si = selCount-1`: Piranha Plant (OAM 5, priority=0, front) rendered first → written to `ScreenBuf1x`
2. `si = ...`: mask sprite (OAM 2, priority=1, behind) rendered after → condition false (BG opaque and priority=1) → skipped
3. Result: Piranha Plant is already in `ScreenBuf1x`, remains visible ❌

### Fix

`PPU.cs`: `RenderSpritesLine()` refactored into three-pass rendering.

```csharp
// After: three-pass rendering
// Each screen pixel records the "winning sprite"'s color and priority, then composited together at end

// Per-pixel buffers
uint* sprColor    = stackalloc uint[256];   // winning sprite color
byte* sprPriority = stackalloc byte[256];   // winning sprite priority
byte* sprSet      = stackalloc byte[256];   // whether an opaque sprite is present

// Pass 2: reverse OAM loop; low-index sprites overwrite high-index → final record is lowest-index winner
for (int si = selCount - 1; si >= 0; si--)
{
    // ... (tile read, flip, clip same as before)
    if (pixel == 0) continue;  // transparent pixels don't compete
    sprSet[screenX]      = 1;
    sprPriority[screenX] = (byte)(priority ? 1 : 0);
    sprColor[screenX]    = NesColors[...];
}

// Pass 3: composite — use winning sprite's priority to decide uniformly
for (int screenX = 0; screenX < 256; screenX++)
{
    if (sprSet[screenX] == 0) continue;
    array_loc = scanOff + screenX;
    // winning sprite priority=1 and BG opaque → entire pixel hidden (including covered high-index sprites)
    if (!ShowBackGround || Buffer_BG_array[array_loc] == 0 || sprPriority[screenX] == 0)
        ScreenBuf1x[array_loc] = sprColor[screenX];
}
```

### Affected Location

`AprNes/NesCore/PPU.cs`: `RenderSpritesLine()` function (complete refactor)

### Fix Result

- Piranha Plant body correctly hidden behind pipe blocks, only visible at pipe opening
- Mushroom power-up correctly hidden inside "?" block/brick, only visible at block edge
- SMB3's mask sprite trick works correctly
