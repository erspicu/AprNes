# BUGFIX22: App Crash / No Warning When AprNesLang.ini Is Missing

**Date**: 2026-02-26

## Problem

When `AprNesLang.ini` language file is not present in the executable directory, the app has two potential issues:

1. **`UpdateSoundMenuText()`** has no guard and directly calls `LangINI.lang_table[lang]["SoundON/OFF"]`. If the language file is absent (`LangLoadOK = false`), this throws a `KeyNotFoundException` → **app crash**
2. **No user notification**: when the language file is missing, the UI silently falls back and the user has no idea what happened; all buttons display their Designer default English text

## Root Cause

### `UpdateSoundMenuText()` has no guard (AprNesUI.cs)

```csharp
// Before — no LangLoadOK check, throws directly when LangINI is not loaded
void UpdateSoundMenuText()
{
    if (_soundMenuItem == null) return;
    _soundMenuItem.Text = NesCore.AudioEnabled
        ? LangINI.lang_table[AppConfigure["Lang"]]["SoundON"]
        : LangINI.lang_table[AppConfigure["Lang"]]["SoundOFF"];
}
```

Both `initUILang()` and `AprNes_ConfigureUI.SetLang()` have `if (!LangINI.LangLoadOK) return;`, but `UpdateSoundMenuText()` was not given the same protection.

### `LangINI` does not record "file missing" state

Previously `LangLoadOK = false` meant both "not yet initialized" and "file missing", making it impossible to distinguish the two externally or display a meaningful notification to the user.

## Fixes

### Fix 1: `LangINI.cs` — Add `LangFileMissing` flag and safe getter helper

```csharp
public static bool LangFileMissing = false;

// Safe getter: if lang not found or key not found, always returns fallback
public static string Get(string lang, string key, string fallback = "")
{
    if (!LangLoadOK) return fallback;
    if (!lang_table.ContainsKey(lang)) return fallback;
    string val;
    return lang_table[lang].TryGetValue(key, out val) ? val : fallback;
}

public static void init()
{
    if (LangLoadOK) return;
    if (!File.Exists(LangFile)) { LangFileMissing = true; return; }
    // ...
}
```

### Fix 2: `AprNesUI.cs` — Startup warning + fix `UpdateSoundMenuText()`

```csharp
// Startup warning
LangINI.init();
if (LangINI.LangFileMissing)
    MessageBox.Show("AprNesLang.ini not found.\nUI will use default text.",
                    "Language file missing", MessageBoxButtons.OK, MessageBoxIcon.Warning);

// UpdateSoundMenuText — add guard + use safe getter
void UpdateSoundMenuText()
{
    if (_soundMenuItem == null || !LangINI.LangLoadOK) return;
    string lang = AppConfigure["Lang"];
    _soundMenuItem.Text = NesCore.AudioEnabled
        ? LangINI.Get(lang, "SoundON",  "Sound ON")
        : LangINI.Get(lang, "SoundOFF", "Sound OFF");
}
```

## Modified Files

| File | Change |
|------|--------|
| `AprNes/tool/LangINI.cs` | Added `LangFileMissing` flag; added `Get(lang, key, fallback)` safe getter helper |
| `AprNes/UI/AprNesUI.cs` | Check `LangFileMissing` on startup and show Warning; added guard to `UpdateSoundMenuText()` and switched to `LangINI.Get()` |

## Behavior

| Situation | Behavior |
|-----------|----------|
| `AprNesLang.ini` exists and loads normally | No change, language loaded normally |
| `AprNesLang.ini` missing | Warning MessageBox on startup, UI keeps Designer default text, no crash |
| Language file exists but key missing | `LangINI.Get()` returns fallback string, no throw |
