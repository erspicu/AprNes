## 36. APU Power-on/Reset Advance + Length Counter Reload Suppression

### Symptom

136 PASS / 38 FAIL. APU power-on/reset and length counter related tests fail:

| Test ROM | Error |
|----------|-------|
| apu_reset/4017_timing | "Frame IRQ flag should be set sooner after power/reset" (#3) |
| apu_reset/4017_written | "At power, $4017 should be written with $00" (#2) |
| blargg_apu_2005/09.reset_timing | Reset timing ($04) |

### Root Cause

#### 1. Power-on/Reset Advance Missing

On real NES, at power-on and reset, the equivalent of `$4017 = $00` is written 9–12 CPU cycles before the first instruction executes. This means the frame counter has already advanced by a few cycles when the CPU begins execution.

The emulator's `initAPU()` and `apuSoftReset()` are called after the reset vector read completes, and `framectrdiv` is set to initial value 7458 with no simulation of this head start.

- `initAPU()` is called during the power-on sequence in Main.cs; `in_tick = true` skipped APU ticking during the reset vector read
- `apuSoftReset()` is called at the end of `ResetInterrupt()` in CPU.cs; the preceding 7 `Mem_r` calls already ticked the APU, but `apuSoftReset()` resets `framectrdiv` to 7458, erasing those ticks

#### 2. Length Counter Reload Suppression Missing

On real NES, when writing to `$4003/$4007/$400B/$400F` on the same cycle as a length clock (half-frame), if counter > 0, the reload is suppressed. The emulator lacks this mechanism.

### Fix

#### 1. APU.cs — initAPU() Power-on Advance

```csharp
// Old:
framectrdiv = 7458;

// New:
framectrdiv = 7450; // 7458 + 2(even jitter) - 10(power-on advance ≈ 9-12 clocks)
```

#### 2. APU.cs — apuSoftReset() Reset Advance

```csharp
// Old:
framectrdiv = 7458;

// New:
framectrdiv = 7450; // match power-on advance
```

#### 3. APU.cs — lengthClockThisCycle Flag Mechanism

Add flag array:
```csharp
static bool[] lengthClockThisCycle = { false, false, false, false };
```

Clear at the start of each `apu_step()` cycle; set in `setlength()` during decrement:
```csharp
// apu_step():
lengthClockThisCycle[0] = lengthClockThisCycle[1] =
lengthClockThisCycle[2] = lengthClockThisCycle[3] = false;

// setlength():
if (!lenctrHalt[i] && lengthctr[i] > 0)
{
    --lengthctr[i];
    lengthClockThisCycle[i] = true;
    ...
}
```

#### 4. APU.cs — apu_4003/4007/400B/400F Reload Suppression

```csharp
// Old:
if (lenCtrEnable[ch])
    lengthctr[ch] = lenctrload[(val >> 3) & 0x1F];

// New:
if (lenCtrEnable[ch] && !(lengthClockThisCycle[ch] && lengthctr[ch] > 0))
    lengthctr[ch] = lenctrload[(val >> 3) & 0x1F];
```
