# BUGFIX 25 – 搖桿設定介面系列修復（Configure UI）

**日期**：2026-03-02  
**版本**：AprNes  
**分類**：Bug Fix

## 問題一：設定介面跳出「非Button類型輸入!」警告

### 現象

開啟搖桿設定介面後，即使未按任何按鍵，也會不斷彈出 `MessageBox` 顯示「非Button類型輸入!」。

### 根本原因

DirectInput 軸值在靜止時存在硬體雜訊（非精確 32767），造成持續產生 axis 事件。  
設定介面的 A/B/Start/Select 欄位只預期 Button 事件，但無篩選機制，軸事件也流入，觸發警告。

### 修復

1. **`joystick.cs`**：加入 `AXIS_NOISE=256`，靠近中心 ±256 範圍內強制歸一到 32767，消除靜止雜訊。
2. **`AprNes_ConfigureUI.cs`**：新增 `ExpectedJoyInputType()` 方法：
   - A/B/Start/Select → 返回 `1`（只接受 Button）
   - UP/DOWN/LEFT/RIGHT → 返回 `2`（接受任何類型）
   - 無焦點 → 返回 `-1`
3. **`AprNesUI.cs`**：設定模式下根據 `ExpectedJoyInputType()` 過濾事件，類型不符則靜默略過。

## 問題二：D-pad D 模式下上下左右無法設定

### 現象

在設定介面點擊「上」欄位後，按下手把 D-pad 上方向，欄位沒有任何反應。

### 根本原因（直接原因）

`Setup_JoyPad_define()` 和 `ExpectedJoyInputType()` 使用 `control.Focused` 判斷目前聚焦的欄位。  
在跨執行緒 `Invoke` + modal `ShowDialog` 的組合下，`Focused` 屬性在 Invoke lambda 執行時可能無法正確反映使用者點擊的欄位（跨執行緒時序競態）。

### 修復（防禦性改進）

新增 `_activeJoyControl` 欄位，透過 `Click`/`Enter` 事件**明確追蹤**使用者最後點擊的欄位，不依賴 `Focused`：

```csharp
TextBox _activeJoyControl = null;

void RegisterJoyActivation()
{
    foreach (TextBox tb in new[] { joypad_A, joypad_B, joypad_START, joypad_SELECT,
                                   joypad_UP, joypad_DOWN, joypad_LEFT, joypad_RIGHT })
    {
        TextBox captured = tb;
        captured.Click += (s, e) => _activeJoyControl = captured;
        captured.Enter += (s, e) => _activeJoyControl = captured;
    }
}
```

`ExpectedJoyInputType()` 和 `Setup_JoyPad_define()` 全部改用 `_activeJoyControl` 比對。

## 問題三：D-pad 方向鍵設定後遊戲方向鍵不動作（Button 模式支援）

### 現象

部分手把 D-pad 在 D 模式下回報為 button event（非 axis），導致設定成功但遊戲無反應（或反之）。

### 修復

- `ExpectedJoyInputType()` 對方向欄位返回 `2`（接受 axis OR button）
- 遊戲路徑（AprNesUI.cs gaming switch）補充：方向按鈕 `button_event==0` 時發 UnPress
