# BUGFIX34: $2007 rendering increment, $2004 read/write during rendering

## 日期: 2026-03-07

## 基線: 174 PASS (舊測試), 107/136 AccuracyCoin PASS (BUGFIX33 後)

## 修復內容

### 1. $2007 Read/Write during rendering - CXinc + Yinc (PPU.cs, MEM.cs)
- **問題**: $2007 讀寫時，VRAM 地址增量一律使用 VramaddrIncrement (+1 或 +32)
- **正確行為**: rendering 期間（visible + pre-render scanlines），$2007 存取應觸發 CXinc() + Yinc()（coarse X + Y increment），而非正常的 +1/+32
- **修復**:
  - PPU.cs 新增 `Increment2007()` helper method
  - 條件: `(ShowBG || ShowSpr) && (scanline < 240 || scanline == 261)` → CXinc + Yinc
  - MEM.cs 所有 ppu_read_fun/ppu_write_fun 中的 vram_addr increment 改用 `Increment2007()`
- **效果**: AccuracyCoin "$2007 Read w/ Rendering" (P16) FAIL → PASS

### 2. $2004 reads during rendering return $FF (PPU.cs)
- **問題**: rendering 期間 CPU 讀取 $2004 返回正常 OAM 值
- **正確行為**:
  - Dots 1-64（secondary OAM clear）: $2004 讀取返回 $FF
  - Dots 257-320（sprite tile loading）: $2004 讀取返回 $FF
  - Dots 65-256（sprite evaluation）: $2004 讀取返回 OAM evaluation latch（未實作）
- **修復**: ppu_r_2004() 加入 scanline/dot 條件檢查，上述兩段回傳 $FF
- **效果**: AccuracyCoin "$2004 Behavior" (P18) err=4 → err=8（tests 1-7 pass, test 8 需 evaluation latch）

### 3. $2004 writes during rendering (PPU.cs)
- **問題**: rendering 期間 $2004 寫入正常寫入 OAM 且 OAMADDR +1
- **正確行為**: 寫入不修改 OAM，OAMADDR 增加 4 並對齊到 4-byte boundary
- **修復**: ppu_w_2004() 加入 rendering 條件（visible + pre-render scanlines），`spr_ram_add = (spr_ram_add + 4) & 0xFC`
- **效果**: 配合 $2004 read fix，tests 6-7 now pass
