## 35. DMC Timer Down-Counter: $4010 Rate Change Does Not Reset the Counter

### Symptom

134 PASS / 40 FAIL. DMC rate tests fail:

| Test ROM | Error |
|----------|-------|
| apu_test/8-dmc_rates | "Rate 0's period is too long" (#3) |
| apu_test/apu_test | Overall FAIL due to test 8 failure |

### Root Cause

The emulator's DMC timer uses an **up-counter** with modulo:

```csharp
// Old:
dmcpos = (dmcpos + 1) % dmcrate;
if (dmcpos == 0) { /* fire */ }
```

Problem: when `$4010` writes change `dmcrate` (e.g. from 54 to 428), the modulo operation immediately uses the new period, effectively "resetting" the current count to the start of the new period.

Real NES DMC timer is a **down-counter**:

1. Timer counts down from the period value
2. When it reaches 0, fires (output unit advances one step) and reloads
3. Writing `$4010` only changes the **reload value**, not the current count

### Impact Analysis

Test ROM `8-dmc_rates` uses `sync_dmc_fast` to sync the DMC timer (rate 54), then switches to the target rate (e.g. rate 0 = 428) and begins playing a 17-byte sample.

**Up-counter behavior** (wrong):
- After sync, dmcpos ≈ 30 (position within rate 54)
- Rate changes to 428; next fire at 428 - 30 = **398 cycles later**
- Total playback time is ~344 cycles too long → "too long"

**Down-counter behavior** (correct):
- After sync, timer remaining ≈ 24 (countdown value at rate 54)
- Rate changes to 428, but countdown continues: 24, 23, ..., 0 → fire
- First fire at **24 cycles later**, then reload to 428
- Total time precisely matches test expectation

### Fix

#### 1. APU.cs — Replace dmcpos with dmctimer

```csharp
// Old:
static int dmcrate = 0x36, dmcpos = 0, ...

// New:
static int dmcrate = 0x36, dmctimer = 0x36, ...
```

#### 2. APU.cs — clockdmc() Changed to Down-Counter

```csharp
// Old:
dmcpos = (dmcpos + 1) % dmcrate;
if (dmcpos == 0)
{
    // fire
}

// New:
if (--dmctimer <= 0)
{
    dmctimer = dmcrate; // reload with current period
    // fire
}
```

#### 3. APU.cs — Update initAPU() Initialization

```csharp
// Old:
dmcrate = dmcperiods[0]; dmcpos = 0;

// New:
dmcrate = dmcperiods[0]; dmctimer = dmcrate;
```

#### 4. apu_4010() Requires No Change

```csharp
// $4010 only changes reload value, does not affect dmctimer
static void apu_4010(byte val)
{
    dmcirq  = (val & 0x80) != 0;
    if (!dmcirq) statusdmcint = false;
    dmcloop = (val & 0x40) != 0;
    dmcrate = dmcperiods[val & 0x0F]; // only changes reload value
}
```

### Timing Verification

Test formula: `delay = period*8*17 - period - 65`

For rate 0 (period=428):
```
After sync_dmc_fast, timer remaining ≈ 24 (countdown at rate 54)
Rate changes to 428, timer continues counting down
After $4015 enable, timer ≈ 18

First fire: ~18 cycles later (reload 428)
Fire 136: 18 + 135*428 = 57798

First $4015 read at: 428*135 - 65 + 4 = 57719
  57798 > 57719 → still playing ✓

Second $4015 read at: 57719 + 90 = 57809
  57798 < 57809 → done ✓
```
