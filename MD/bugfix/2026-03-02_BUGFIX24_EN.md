# BUGFIX 24 – Replace WinMM Joystick API with DirectInput8 + XInput

**Date**: 2026-03-02
**Version**: AprNes
**Category**: Refactor / Bug Fix

---

## Motivation

The original joystick input API used WinMM (`joyGetPos` / `joyGetDevCaps`), which is too outdated:
- Supports only up to 6 axes + a limited number of buttons
- Cannot correctly identify XInput gamepads (Xbox series)
- No native D-pad / POV hat support

Refactoring goal: use **DirectInput8** (for non-XInput gamepads) + **XInput** (for Xbox/XInput gamepads) in dual mode.

---

## Issue 1: AccessViolationException (DirectInput COM Interop)

### Symptom

```
System.AccessViolationException: Attempted to read or write protected memory.
  at NativeTools.IDirectInput8W.EnumDevices(...)
  at NativeTools.DirectInputNative.EnumJoysticks(...)
```

### Root Cause

When using `[ComImport]` + `[InterfaceType(ComInterfaceType.InterfaceIsIUnknown)]` to create a COM RCW, the CLR mishandles marshaling a managed delegate (`DIEnumDevicesCallback`) as a COM method parameter. A memory access violation occurs before the call enters the native vtable via `EnumDevices`.

### Fix

Completely removed the `[ComImport]` interface and switched to **raw vtable dispatch**:

```csharp
// Read vtable
IntPtr vtable = Marshal.ReadIntPtr(comPtr);
IntPtr fnPtr  = Marshal.ReadIntPtr(vtable, slot * IntPtr.Size);
var fn = Marshal.GetDelegateForFunctionPointer(fnPtr, typeof(T)) as T;

// callback passed as IntPtr, bypassing CLR delegate marshal
IntPtr cbPtr = Marshal.GetFunctionPointerForDelegate(cb);
fn(pDI, DI8DEVCLASS_GAMECTRL, cbPtr, IntPtr.Zero, DIEDFL_ATTACHEDONLY);
GC.KeepAlive(cb);
```

## Refactoring Details

### New: `AprNes/tool/DirectInputHelper.cs`

- `DIJOYSTATE` (80 bytes), `DIPROPRANGE`, `DIDATAFORMAT`, `DIOBJECTDATAFORMAT` struct definitions
- vtable slot constants (`DI8Slot`, `DI8DevSlot`)
- 9 `[UnmanagedFunctionPointer(CallingConvention.StdCall)]` delegate types
- `DirectInputNative` static class: `CreateDirectInput()` / `EnumJoysticks()` / `OpenDevice()` / `PollDevice()` / `ReleaseDevice()`
- `DfDIJoystick` static format object (corresponding to `DIJOYSTATE`)
- SetupDi scan for `IG_` paths to exclude XInput devices from DI enumeration

### Modified: `AprNes/tool/joystick.cs`

- Removed WinMM polling; replaced with DirectInput + XInput dual-path
- `Init(IntPtr hwnd)` initializes DI and XInput
- `joy_event_captur()` outputs standard `joystickEvent` list
- POV hat → X/Y conversion (`PovToXY()`), values fixed at 0/32767/65535
- AXIS_NOISE=256 to filter center noise

### Modified: `AprNes/tool/NativeAPIShare.cs`

- Removed WinMM `joyGetPos` / `joyGetDevCaps` P/Invoke and related structs
