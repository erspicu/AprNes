# PAL vs NTSC 顯示比例 (Pixel Aspect Ratio)

> 資料來源：Gemini 知識庫查詢 (2026-03-29)

## 像素寬高比 (PAR)

| | NTSC (2C02) | PAL (2C07) |
|---|---|---|
| **PAR** | **8:7** (1.1428) | **≈25:18** (1.3889) |
| 像素特性 | 略寬於正方形 | 明顯寬於正方形 |

### NTSC PAR 推導

- NTSC CRT 方形像素 dot clock = `135/11 MHz` (~12.2727 MHz)
- NES pixel clock = `1.5 × f_sc` = `472.5/88 MHz` (~5.3693 MHz)
  - f_sc = NTSC 色彩副載波 = 315/88 MHz
- PAR = (135/11) ÷ (472.5/88) = 11880/10395 = **8/7**

### PAL PAR 推導

- PAL CRT 方形像素 dot clock = `96/13 MHz` (~7.3846 MHz)
  - 基於 52µs 活躍行時間
- PAL NES pixel clock = ~5.3203 MHz (2C07)
- PAR = (~7.3846) ÷ (~5.3203) ≈ **1.388** ≈ 25/18

## 垂直顯示差異

| | NTSC | PAL |
|---|---|---|
| NES 輸出行數 | 240 | 240 |
| CRT 可見行數 | ~240 | ~288 |
| 垂直填充率 | ~100% | ~83.3% (240/288) |
| 黑邊 | 無 | 上下各 ~24 行 |

PAL NES 只輸出 240 行，但 PAL CRT 有 288 行可見區域，造成上下各約 24 行的黑邊 (letterbox)。

## 正確顯示寬高比 (DAR) — 256×240 渲染

| | NTSC | PAL |
|---|---|---|
| **DAR** | **128:105** (~1.22:1) | **≈3:2** (~1.48:1) |
| 計算 | 256 × (8/7) / 240 = 1.219 | 256 × (25/18) / 240 = 1.481 |

- NTSC: 畫面略窄於 4:3 (1.333)
- PAL: 畫面明顯寬於 4:3，因為 240 行被壓縮在 288 行空間中

### 整數縮放建議

- NTSC: 寬度 ×8, 高度 ×7 (2048×1680)
- PAL (含黑邊 256×288): 整體 DAR ~1.23:1，但內部 256×240 遊戲畫面仍為 1.48:1

## 模擬器實作備註

- 一般模式：維持 1:1 像素或簡單 4:3 拉伸即可，不需特別處理
- 類比模式：應根據 Region 套用正確 PAR，PAL 需考慮黑邊渲染
- PAL 遊戲在真實 CRT 上角色看起來較矮胖，這是正常的硬體特性
