AprNes

C# 實作的任天堂紅白機模擬器，以 cycle-accurate 精準度為目標，通過業界最嚴格的測試驗證， 並運用 AI 協作持續演進。

⭐ GitHub 下載最新版 查看測試報告

核心特色

從硬體文件出發，逐一實作每個 NES 子系統的精確行為

⏱️
Cycle-Accurate 時序

CPU、PPU、APU 全部以 per-cycle 精度模擬。每次記憶體存取觸發精確的 tick 計時，NMI/IRQ 延遲、DMA 週期偷取等細節均符合真實硬體行為。

🔀
完整 DMA 模擬

OAM DMA 514 cycle、DMC 週期偷取（3–4 cycle）、phantom read、implicit abort 等所有 DMA 時序邊界情況均正確實作，通過最嚴苛的 DMA 測試 ROM。

🎵
APU 五聲道音效

Pulse×2、Triangle、Noise、DMC 完整實作。自製 WaveOut 輸出（winmm.dll），44100Hz 16-bit，無任何第三方音效套件依賴。

🤖
AI 協作開發

以 AI 輔助研究 NES 硬體行為、分析測試 ROM 失敗原因、設計修復方案、效能最佳化。探索 AI-driven 嵌入式系統模擬開發的可行邊界。

🧪
自動化測試流程

174 個 blargg test ROM + 136 個 AccuracyCoin 測試全自動化。截圖比對、CRC 驗證、headless 批次執行，每次 commit 均有正確性保障。

⚡
效能最佳化研究

從 181 FPS 提升至 ~273.8 FPS（+50.7%）。19 項成功最佳化，涵蓋 unsafe pointer、函式指標表、JIT AggressiveInlining、early-exit guard、Release 組態 loop unroll、switch→if/else（CPU/PPU/IO）等技巧。

🗺️
多 Mapper 支援

支援 Mapper 0/1/2/3/4（MMC3）/5（MMC5）/7/11/66/71，涵蓋市場上約 90% 的 NES 遊戲。MMC3 含完整 IRQ A12 上升沿時序。

🖼️
畫面 Scaler Filter

xBRZ 2x/4x/6x 高品質影像放大濾鏡，多核心平行加速。內建 NTSC CRT 比例修正與 YIQ color bleeding 模擬效果。

開發歷史

從業餘學習到 cycle-accurate 精準度的演進歷程

2015
重啟舊專案
將未完成的 NES 模擬器重新啟動，目標是以學習和研究為主，理解 NES 硬體架構。
2016.10
首版發布 — Mapper 0
第一個公開版本，支援 NROM (Mapper 0)。Sprite0Hit、SpriteOverflow flag 已實作，sprite/背景穿透正確。
2016.11
Mapper 擴充 + 介面完善
加入 MMC1 (M1)、UNROM (M2)、CNROM (M3) 支援；SRAM 存檔；ZIP 讀取；介面設定項目。
2016.11.21
MMC3 + xBRZ Filter
MMC3 (Mapper 4) 支援，整體遊戲相容度約 70%。xBRZ 2x/4x/6x 影像濾鏡加入，多核心加速。
2017.02
Scanline Filter + 暫停更新
加入 NTSC Scanline Filter、YIQ color bleeding 效果，移除所有第三方依賴（單一 167KB EXE）。 版本暫停："I will be back."
2026.02
AI 輔助復活 — APU + 精準時序
以 Claude AI 協作全面重建：APU 五聲道音效完成、VBL/NMI 精確時序、CPU interrupt 修正、 Sprite timing 修正，通過 blargg 全部測試。
2026.03
Cycle-Accurate 達成 — 效能最佳化
AccuracyCoin 136/136 滿分達成（DMA timing、per-dot sprite eval、PPU rendering 精準修正）。 效能最佳化 +50.7%（181→273.8 FPS）。
